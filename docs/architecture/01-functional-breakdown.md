# YFI 工业 AI 小模型平台功能拆解

## 1. 项目目标与 MVP 闭环

本项目目标是建设面向 YFI 及相关 BU 的工业 AI 小模型平台，在安全可控的私有化环境中统一管理工业数据、标注、训练、模型、推理服务、边端部署和资源治理能力。

6 个月 MVP 聚焦一条可运行的闭环：

```text
数据接入/数据集管理 -> 数据标注/审核 -> 训练/调优 -> 模型评估/注册 -> 推理服务/边端下发 -> 权限/审计/资源治理
```

MVP 必须证明以下能力：

- 组织权限、SSO、RBAC/ABAC 基线可用。
- 数据集元数据、版本、权限申请和对象存储链路可用。
- 标注任务可创建、分配、审核，并集成成熟标注工具。
- 训练任务可提交、运行、查看日志和指标，并产生模型产物。
- 模型可注册、版本化、评估和检索。
- 模型可通过中心端推理服务部署，并可下发到边端服务器。
- 资源配额、基础监控和审计日志贯穿主要操作。

## 2. 用户角色

| 角色 | 核心职责 | MVP 关注点 |
| --- | --- | --- |
| 超级管理员 | 平台全局配置、集团级组织、权限策略、公共资源和安全策略 | 组织树、全局角色、平台配置、审计查看 |
| BU 子管理员 | 管理 BU 内用户、角色、数据集、模型、镜像和资源申请 | BU 资源、用户授权、数据/模型审批 |

| 数据审核工程师 | 审核标注结果，保障数据集质量 | 审核任务、退回、通过、质量记录 |
| 模型训练工程师 | 算法选择、训练调优、评估验证、模型工程化 | 训练任务、日志、指标、产物注册 |
| 模型应用工程师 | 模型导出、部署、推理服务和边端服务器管理 | 模型部署、服务状态、边端下发 |
| 平台运维/资源管理员 | 集群、资源池、存储池、监控、告警、运行稳定性 | 资源配额、节点监控、存储用量、告警 |

## 4. MVP 范围边界

### 4.1 MVP 明确包含

- 平台门户、组织、用户、角色、权限和审计。
- 数据源登记、数据集上传、元数据、版本、权限申请和对象存储。
- 标注任务、标签模板、标注工具集成、审核流。
- 训练任务、调优任务、日志、指标和产物管理。
- 模型注册、模型版本、评估指标和模型权限。
- KServe 中心端推理服务部署。
- 边端服务器登记、模型下发、状态回传。
- 资源配额、基础监控、存储用量和运行审计。

### 4.2 MVP 明确不包含

- 全量工业协议适配。
- 完整自研多模态标注器。
- 自动超参调优高级能力。
- 量化、剪枝、蒸馏自动流水线。
- 复杂数据增强插件市场。
- 多区域灾备和高级安全围栏。

## 5. 待确认事项

- 企业 SSO/IAM 类型、协议和测试账号。
- GPU/NPU 型号、驱动、device plugin 和资源池隔离要求。
- 数据安全分级、内容安全规则和审计保存周期。
- 目标存储容量、单数据集大小、模型包大小和增长率。
- 边端服务器操作系统、网络连通方式和离线部署要求。
- RTO/RPO、备份策略、灾备等级和上线 SLA。
