# SMP 工业 AI 小模型平台整体理解

> 状态：重建基线理解文档
> 来源：`docs/business/`、`docs/prototype/`
> 日期：2026-05-15

## 1. 项目一句话定位

SMP 是面向工业场景的小模型全生命周期管理平台，核心价值是把数据、标注、训练、评估、部署、边端下发、监控告警、权限审计与报表运营统一到一个可治理的平台中。

## 2. 业务闭环

```text
组织/用户/权限
  -> 数据源接入
  -> 数据集与版本管理
  -> 数据加工 Pipeline / 算子
  -> 标注任务 / 标注审核
  -> 训练任务 / 实验管理
  -> 模型评估 / 模型市场
  -> 在线推理 / 批量推理
  -> 边端服务器模型下发
  -> 监控告警 / 审计 / 报表
  -> 推理结果与日志回流
```

## 3. 五大领域

### 3.1 DATA 数据域

权威资料：

- `docs/business/bizdocs/03-01-系统功能-数据管理.md`
- `docs/business/domain/01-领域对象-数据域.md`
- `docs/business/rules/01-数据管理规则.md`

核心对象包括数据源、数据集、数据集版本、标注任务、标注结果、数据血缘、Pipeline、算子、访问申请。关键规则是数据入平台前置安全检测、受限数据访问审批、已发布版本不可变、跨 BU 授权、删除前引用检查。

### 3.2 MODEL 模型域

权威资料：

- `docs/business/bizdocs/03-02-系统功能-模型开发.md`
- `docs/business/domain/02-领域对象-模型域.md`
- `docs/business/rules/02-模型开发规则.md`

核心对象包括开发环境、训练任务、实验、模型、模型版本、模型评估、模型工程化任务。关键规则是训练必须引用已激活数据集、提交前检查资源配额、模型评估通过后才能发布、活跃推理部署的模型版本不得删除。

### 3.3 INFERENCE 推理域

权威资料：

- `docs/business/bizdocs/03-03-系统功能-模型部署.md`
- `docs/business/domain/05-领域对象-推理域.md`
- `docs/business/rules/03-推理部署规则.md`

核心对象包括推理服务、服务版本、流量策略、批量推理任务、边端服务器、边端部署、第三方推理服务。关键规则是仅已发布/审批模型可部署，流量权重总和必须等于 100%，健康检查失败自动回滚，边端下发需审批与文件完整性校验。

### 3.4 RESOURCE 资源域

权威资料：

- `docs/business/domain/03-领域对象-资源域.md`
- `docs/business/rules/04-资源管理规则.md`

核心对象包括集群、节点、资源池、资源配额、存储池、容器镜像和 AI 芯片。关键规则是 BU 不得超过配额，GPU/NPU 训练任务队列调度，基础镜像必须安全扫描，单任务最大资源申请不得超过资源池上限。

### 3.5 PLATFORM 平台域

权威资料：

- `docs/business/bizdocs/03-04-系统功能-平台管理.md`
- `docs/business/domain/04-领域对象-平台域.md`
- `docs/business/rules/05-平台与权限规则.md`

核心对象包括租户/组织、用户、角色、权限、审计日志、通知、系统配置、报表。关键规则是 tenantId 隔离、用户至少一个角色、最后一个超级管理员不可删除/停用、连续登录失败锁定、审计日志不可修改删除且至少保留 3 年。

## 5. 架构与实现含义

1. 主后端应以领域模块组织，而不是按原型页面堆 Controller。
2. 所有高风险操作必须有权限校验、状态机校验和审计事件。
3. 数据集、模型、推理服务、边端部署等核心资源必须具备 owner、tenantId、version/status、权限、审计记录。
4. AI/MLOps 外部系统通过 `ai-adapter/` 隔离，主后端保持业务事实源。
5. 大文件进入对象存储，数据库仅存元数据、对象 key、hash、状态、权限和审计关联。
6. 前端应从 `docs/prototype/` 复刻信息架构，再逐步接入真实 API；不得以静态 mock 代替核心业务流验收。
