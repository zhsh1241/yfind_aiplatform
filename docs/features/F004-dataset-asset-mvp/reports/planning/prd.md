﻿# PRD: F004-dataset-asset-mvp

## Problem

当前平台已经具备基础骨架、权限基线与可点击原型，但缺少一个正式的数据资产中心。没有这个能力，后续标注、训练、模型、推理等功能都无法围绕统一的数据集版本开展工作，也无法建立清晰的数据授权与追踪边界。

## Goals

- 建立数据资产门户，支持列表、搜索、筛选、详情与图片样例预览。
- 建立平台内数据接入能力，支持本地上传、元数据登记、版本生成。
- 建立基础治理能力，支持权限申请、审批、数据集级查看授权与版本级下载授权。
- 建立基础处理链路，支持 hash、去重策略与异步预处理队列。
- 建立存储抽象 seam，首版默认本地文件系统，实现后续切换对象存储的可扩展性。

## Users

- 平台管理员：查看全量数据集、审批权限申请、审计关键操作。
- 数据集所有者 / 业务负责人：创建数据集、上传文件、发布新版本、审批他人访问申请。
- 标注人员 / 算法工程师：检索可见数据集、申请访问、下载获批版本、查看图片样例。
- 后续 AI coding agents：围绕数据集版本继续实现标注、训练、模型等能力。

## Functional Requirements

- 后端支持数据集、数据集版本、文件条目、授权申请、审批记录、异步处理任务等基础模型。
- 平台支持创建数据集并维护元数据。
- 平台支持本地文件上传，并为上传文件记录 hash、大小、类型、来源与版本归属。
- 平台支持基于上传结果生成数据集版本。
- 平台支持列表、搜索、筛选与详情页。
- 平台支持图片样例预览；非图片文件允许上传，但可退化为不支持预览。
- 平台支持数据集级查看权限控制。
- 平台支持版本级下载权限控制。
- 平台支持权限申请与审批流。
- 平台支持去重策略与异步预处理队列，处理状态可查询。
- 后端存储层通过抽象接口隔离，首版使用本地文件系统实现。

## Non-goals

- 外部系统同步。
- 对象存储引用能力作为 F004 首批正式能力。
- 数据源连接测试能力。
- 通用文档/表格/音视频等完整在线预览。
- 生产级对象存储接入、跨云存储策略、冷热分层等高级存储方案。
- 高级数据质量规则引擎、自动标签识别、复杂工作流编排。

## Product Decisions

- 范围采用均衡 MVP，不做单点演示版。
- 查看与下载采用分层授权：数据集级查看、版本级下载。
- 上传类型采用“通用文件可上传，图片预览有保证”的策略。
- 存储实现采用“抽象接口 + 默认本地文件系统”的方式，避免 F004 被对象存储环境前置阻塞。
- 预处理采用异步任务模型，失败不影响原始数据与版本记录保留。

## Reuse Strategy

### Must Reuse

- `backend/` 既有 Spring Boot 单体骨架与 F002 权限边界。
- `frontend/` 既有 React + TypeScript + Ant Design 基线。
- F002 已建立的身份、组织、权限与审计边界。
- `tools/ai-scaffold` 与 `docs/features/` 既有 feature 交付流程。
- F003 原型中的数据资产/权限交互概念，作为前端信息架构参考而不是复制实现。

### Duplication Rejected

- 不创建第二个独立数据服务；数据资产能力应落在主 `backend/` 中。
- 不在 `ai-adapter/` 中复制数据元数据与权限逻辑。
- 不复制第二套上传网关或第二套授权申请流程。
- 不为对象存储预建一套与本地文件系统完全平行的业务流程。

### Approved New Seams

- 允许新增数据集、版本、文件条目、授权申请、审批、异步处理任务等领域模块，因为当前仓库尚无这些业务边界。
- 允许新增存储接口抽象，因为未来对象存储接入是已知方向，而当前需要先用本地文件系统落地。
- 允许新增预处理任务状态与执行 seam，因为用户明确要求 F004 就纳入异步处理链路。

## Success Criteria

- 计划文档明确范围、边界、风险、复用策略与验收方向。
- 后续 build-feature 阶段可围绕以下链路实现并验收：创建数据集 -> 上传文件 -> 生成版本 -> 预览图片 -> 发起申请 -> 审批 -> 下载版本。
- 后续 build-feature 阶段需证明上传 / 版本 / 去重 / 异步处理链路稳定。
- 存储抽象与授权分层在 contract / test-plan 中被正式冻结。

## Risks

- 异步预处理队列若设计过重，可能拖慢 F004 交付节奏。
- 数据集级查看与版本级下载的分层授权若契约不清，会导致前后端行为不一致。
- 通用文件上传与图片预览并存时，文件类型判断与错误处理需要明确约束。
- 本地文件系统实现若不提前抽象存储接口，后续切换对象存储成本会显著提高。

## Open Questions

- F004 的异步预处理首版最少做哪些动作：仅元数据抽取、缩略图生成、还是额外校验？
- 去重策略是默认阻止重复上传，还是允许覆盖/跳过/保留副本？
- 数据集版本状态是否需要区分草稿、处理中、可用、已归档？
- 审批流首版是一层审批还是支持多角色审批？
