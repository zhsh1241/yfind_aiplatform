## 1. 整体部署拓扑

SMP 平台采用**中心-边端**混合部署架构，中心端承载数据管理、模型训练、中心推理等核心服务；边端承载本地推理和数据采集。

### 1.1 逻辑架构层次

```
┌─────────────────────────────────────────────────────────────────┐
│                       访问层（Access Layer）                      │
│          Web Browser / IDE Extension / REST API / gRPC          │
└─────────────────────────┬───────────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────────┐
│                    API 网关层（Gateway Layer）                     │
│         认证鉴权 / 限流 / 路由 / SSL 终止 / 审计日志              │
└────────┬──────────────┬───────────────────┬─────────────────────┘
         │              │                   │
┌────────▼─────┐ ┌──────▼──────┐ ┌──────────▼──────────┐
│   数据服务层  │ │   模型服务层  │ │    推理服务层         │
│ DATA Domain  │ │ MODEL Domain │ │  INFERENCE Domain   │
│              │ │              │ │                     │
│ - 数据源管理  │ │ - 训练调度   │ │ - 在线推理服务       │
│ - 数据加工   │ │ - 模型存储   │ │ - 批量推理引擎       │
│ - 数据标注   │ │ - 开发环境   │ │ - A/B流量控制       │
│ - 数据集管理  │ │ - 评估工具   │ │ - 推理网关          │
└────────┬─────┘ └──────┬──────┘ └──────────┬──────────┘
         │              │                   │
┌────────▼──────────────▼───────────────────▼──────────┐
│                资源管理层（Resource Layer）              │
│    Kubernetes（GPU/NPU 节点池）/ YARN / 物理主机         │
│    存储：对象存储（OSS/S3）/ 分布式文件系统（HDFS/Ceph）  │
└──────────────────────┬───────────────────────────────┘
                       │
┌──────────────────────▼───────────────────────────────┐
│                 平台管理层（Platform Layer）             │
│    认证（SSO）/ RBAC / 多租户隔离 / 审计 / 通知          │
└──────────────────────────────────────────────────────┘
```

## 2. 计算资源部署规格（暂定）

| 节点类型 | 数量（暂定） | 规格 | 用途 |
|---------|-----------|------|------|
| 管理节点 | 3（HA） | 16C/64G | Kubernetes Master / 数据库 / 消息队列 |
| GPU 训练节点 | ≥4 | 8×A100 80G | 模型训练 / 微调 |
| CPU 推理节点 | ≥2 | 32C/128G | 在线推理（CPU 模型） |
| GPU 推理节点 | ≥2 | 4×A100 | 在线推理（GPU 加速） |
| 存储节点 | ≥3 | 1TB NVMe + 大容量 HDD | 模型文件 / 数据集存储 |

> **待确认**：实际规格需根据并发用户数和训练任务规模测算，详见 `docs/bizdocs/06-非功能性需求.md` NFR-P01~P03。

## 3. 网络规划（暂定）

| 网段 | 用途 | 协议 |
|-----|-----|------|
| 管理网 | 节点间控制面通信 | 万兆以太网 |
| 训练网（RDMA） | GPU 间高速互联（分布式训练） | InfiniBand / RoCE |
| 存储网 | 数据集/模型文件读写 | 万兆以太网 / NVMe-oF |
| 业务网 | 用户访问 / API 网关 | HTTPS（TLS 1.3） |
| 边端网 | 中心-边端通信 | HTTPS / mTLS |

## 4. 高可用与容灾（待补充）

> 以下内容为框架占位，具体 RTO/RPO 目标见 `docs/bizdocs/06-非功能性需求.md` NFR-A01~A03，确认后补充此节。

- **中心端 HA**：Kubernetes 多 Master（3节点 etcd 集群）；数据库主从复制（`待确认：MySQL / PostgreSQL / 国产数据库`）
- **推理服务 HA**：多副本部署 + HPA；跨 AZ 部署（`待确认：单 DC 还是双活 DC`）
- **边端容灾**：边端 Agent 本地缓存（2GB 日志缓冲，见 INF-011），网络中断时本地运行不中断
- **数据备份**：数据集/模型文件定期快照，`待确认：备份周期和保留策略`
