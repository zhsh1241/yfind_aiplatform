# 05 平台与权限规则

> **覆盖域**：平台域（PLT / SYS）——多租户隔离、用户认证、角色权限、账号安全、审计日志、密码策略、SSO、BU 管理员权限边界。  
> **规则总数**：14 条

---

### PLT-001 所有数据访问以 tenantId 为边界，跨租户访问须显式授权

- **等级**：MUST
- **触发场景**：任何用户或服务账号发起数据集、模型、任务、用户等资源的查询或操作请求时
- **规则描述**：平台所有业务对象（数据集、模型、训练任务、推理服务、审计日志等）均关联 tenantId；系统在所有查询层自动注入 tenantId 过滤条件，用户默认只能访问本租户（BU）内的资源；跨租户访问须持有显式授权记录，否则操作被拒绝。跨租户访问行为作为 CRITICAL 级审计事件记录。
- **校验逻辑**：认证成功后，系统从 JWT Token 中提取 tenantId，并在所有数据库查询中自动附加 `WHERE tenant_id = {tenantId}` 过滤；API 层在请求到达业务逻辑前校验 tenantId 一致性；超级管理员跨租户操作须显式指定目标 tenantId 并记录原因。
- **异常处理**：跨租户访问无授权时，返回 404（不暴露资源存在性）；发现跨租户访问尝试时，记录 CRITICAL 级审计日志并触发安全告警，通知安全管理员。
- **配置项**：超级管理员跨租户操作是否需要二次确认（默认需要）；跨租户访问授权有效期（默认 30 天）
- **交互终端**：Interface / 系统自动

---

### PLT-002 用户必须持有至少一个角色，零角色状态为无效状态

- **等级**：MUST
- **触发场景**：管理员撤销用户角色操作时；用户最后一个角色到期或被删除时；角色批量变更操作时
- **规则描述**：平台用户在任意时刻必须持有至少一个有效角色（在某个 BU 内）；系统不允许操作导致用户进入零角色状态；角色撤销操作在执行前须校验用户剩余有效角色数量。
- **校验逻辑**：执行角色撤销操作前，系统统计该用户在所有 BU 内的有效角色数量；若撤销后总数将为 0，则拒绝操作；角色到期（临时权限）时，若到期后用户将进入零角色状态，系统提前 7 天通知管理员补充角色配置。
- **异常处理**：校验失败时，系统提示"操作后用户将无任何角色，请先为该用户分配新角色后再撤销现有角色"；操作被阻断；若因系统异常导致用户进入零角色状态，系统自动为其分配"默认只读角色"并告警管理员修正。
- **配置项**：零角色保护是否可由超级管理员豁免（默认不可豁免）；临时角色到期预警提前天数（默认 7 天）
- **交互终端**：PC / Interface

### PLT-003 系统内最后一个超级管理员不得被删除或停用

- **等级**：MUST
- **触发场景**：超级管理员执行删除或停用另一个超级管理员账号操作时；批量停用用户操作中涉及超级管理员时
- **规则描述**：平台系统中必须始终存在至少一个有效的超级管理员账号；当系统内仅剩最后一个超级管理员时，任何删除或停用该账号的操作均被系统拒绝，防止系统失去管理入口。
- **校验逻辑**：删除或停用操作发起时，系统统计当前处于"活跃"状态且持有 SUPER_ADMIN 角色的用户数量；若数量 = 1 且目标用户即为该超级管理员，则拒绝操作。
- **异常处理**：校验失败时，系统提示"无法执行此操作，系统中仅剩一位超级管理员，请先提升其他用户为超级管理员后再操作"；操作被完全阻断，不允许任何绕过方式。
- **配置项**：超级管理员最小保留数量（默认 1，可调整为更高值以提升安全性）
- **交互终端**：PC / Interface

### PLT-004 连续登录失败 5 次后账号锁定 30 分钟

- **等级**：MUST
- **触发场景**：用户在登录页面连续输入错误密码时；API 接口连续认证失败时
- **规则描述**：用户账号在连续登录失败 5 次后，系统自动将账号锁定 30 分钟；锁定期间该账号无法登录（包括正确密码也不允许登录）；30 分钟后自动解锁，或由管理员手动提前解锁。（来源：`04-领域对象-平台域.md` User.lockedUntil / failedLoginCount 字段定义；`05-01-系统功能-平台基础.md` FUNC-PLT-001 约束）
- **校验逻辑**：每次登录失败后，系统将 `failedLoginCount` 加 1；当 `failedLoginCount >= 5` 时，将账号状态置为 LOCKED，设置 `lockedUntil = 当前时间 + 30 分钟`；登录请求到达时，系统先检查账号是否处于锁定状态及 `lockedUntil` 是否未过期；若锁定则拒绝登录，返回锁定剩余时间。
- **异常处理**：锁定期间登录尝试时，系统提示"账号已被锁定，请 [剩余分钟数] 分钟后重试或联系管理员解锁"，不展示具体锁定原因（防止信息泄露）；锁定事件记录审计日志；超级管理员账号锁定时，系统额外发送告警通知给其他超级管理员。
- **配置项**：连续失败锁定触发次数（默认 5 次，BU 级可配置）；账号锁定时长（默认 30 分钟，BU 级可配置）；管理员手动解锁权限（BU 管理员可解锁本 BU 用户）
- **交互终端**：PC / Interface / 系统自动

### PLT-005 审计日志不可修改或删除，写入后永久只读

- **等级**：MUST
- **触发场景**：任何用户或系统进程尝试修改、删除审计日志记录时；审计日志系统接收写入请求时
- **规则描述**：审计日志（AuditLog）一经写入，不允许任何角色（包括超级管理员）执行修改或删除操作；审计日志仅支持追加写入和查询读取；审计日志的完整性通过 SHA-256 签名机制保障，可对任意条目进行签名验证。（来源：`04-领域对象-平台域.md` AuditLog 不可变性约束）
- **校验逻辑**：审计日志服务的 API 层仅开放 `recordOperation`（写入）和 `queryAuditLogs`（查询）接口；DELETE 和 UPDATE 接口在系统层面不存在；任何尝试直接操作审计日志存储的数据库操作通过 DB 权限管控拦截；日志写入时计算 SHA-256 签名并随日志一并存储，查询时可调用 `verifySignature` 验证。
- **异常处理**：检测到针对审计日志的修改/删除尝试时（如直接 DB 操作），系统记录 CRITICAL 级安全事件，通知安全管理员，并触发完整性检查流程；应用层拒绝操作，返回 403。
- **配置项**：无（不可配置，为绝对约束）
- **交互终端**：Interface / 系统自动

### PLT-006 审计日志最少保留 3 年

- **等级**：MUST
- **触发场景**：审计日志生命周期管理任务执行时；存储清理策略配置时
- **规则描述**：所有审计日志的最低保留期限为 3 年；系统不允许配置少于 3 年的审计日志自动清理策略；超过 3 年的日志可在满足合规要求后按需清理，但清理操作本身须生成新的审计记录。（来源：`04-领域对象-平台域.md` AuditLog 数据保留周期"至少 3 年"）
- **校验逻辑**：配置审计日志生命周期策略时，系统校验 `retentionPeriod >= 3 * 365 天`；低于 3 年的配置值被拒绝；自动清理任务仅清理 `timestamp < 当前时间 - retentionPeriod` 的记录，且清理操作本身写入新的审计日志。
- **异常处理**：配置保留期低于 3 年时，系统提示"审计日志保留期不得少于 3 年（1095 天），当前设置为 [N] 天"，拒绝保存；合规清理超过保留期的日志时，系统自动生成"批量清理审计日志"的操作记录。
- **配置项**：审计日志保留期（最小值 3 年 = 1095 天，默认 3 年，可延长）；日志存储介质（冷存储 / 归档存储）
- **交互终端**：PC（安全管理员配置）/ 系统自动

### PLT-007 密码必须满足复杂度要求，策略按租户可配置

- **等级**：CONFIG
- **触发场景**：用户设置初始密码时；用户修改密码时；管理员重置用户密码时
- **规则描述**：本地账号密码必须满足平台或 BU 配置的复杂度要求（最小长度、大小写、数字、特殊字符、不可与历史密码相同等）；密码复杂度策略可按租户（BU 级）独立配置，以适应不同 BU 的安全策略；SSO 认证用户不设置本地密码，不受此规则约束。（来源：`05-01-系统功能-平台基础.md` FUNC-PLT-001 / FUNC-PLT-005 约束）
- **校验逻辑**：密码设置或修改时，系统按该用户所属 BU 的密码策略校验：①长度 ≥ min 值；②包含大写字母（若配置要求）；③包含数字（若配置要求）；④包含特殊字符（若配置要求）；⑤不与最近 N 次历史密码相同；任一不满足则拒绝。
- **异常处理**：密码不符合复杂度要求时，系统提示具体未满足的项目（如"密码长度不足，最少需要 8 位"），操作被阻断；不暴露历史密码比对的具体结果（仅提示"密码不可与近期使用的密码相同"）。
- **配置项**：密码最小长度（BU 级可配置，建议默认 8 位）；是否要求大写字母（默认是）；是否要求数字（默认是）；是否要求特殊字符（默认是）；历史密码禁止复用次数（默认 3 次）；密码有效期（默认 90 天，到期前 7 天提醒）
- **交互终端**：PC / Interface

### PLT-008 SSO 用户遵循 IDP 会话超时策略，不支持密码重置

- **等级**：MUST
- **触发场景**：通过 SSO（OAuth2 / SAML2）认证登录的用户进行会话管理时；SSO 用户尝试修改密码或重置密码时
- **规则描述**：SSO 认证用户的会话有效期遵循企业 IDP（身份提供商）下发的 Token 有效期和会话超时策略；平台不覆盖或延长 IDP 的会话超时设置；SSO 用户在平台上无本地密码，不支持密码找回和密码重置功能。（来源：`05-01-系统功能-平台基础.md` FUNC-PLT-002 约束；FUNC-PLT-005 约束）
- **校验逻辑**：SSO 用户登录后，平台 JWT Token 有效期设置为不超过 IDP Token 有效期；IDP Token 失效时，平台 Token 同步失效，用户须重新通过 IDP 认证；密码重置接口在接收 SSO 用户请求时，检查 `authType = SSO` 后直接拒绝操作。
- **异常处理**：SSO 用户尝试密码重置时，系统提示"SSO 账号密码由企业 IDP 管理，请通过企业门户修改密码"；IDP 服务不可用时，SSO 用户无法登录，系统提示"单点登录服务暂时不可用，请联系 IT 部门"，同时启用本地账号作为应急登录入口（若已配置）。
- **配置项**：SSO 首次登录是否自动创建本地账号（默认是，可配置为需手动审核）；IDP 不可用时是否启用本地账号紧急登录（默认是）；企业 IDP 对接 YF LDAP（ARC-01 已确认）
- **交互终端**：PC / Interface / 系统自动

### PLT-009 BU 管理员权限边界限定在本 BU 子树内

- **等级**：MUST
- **触发场景**：BU 管理员执行用户管理、资源分配、角色授权等操作时
- **规则描述**：BU 管理员（BU_ADMIN 角色）只能管理本 BU 及其下属项目内的用户、资源和配置；不得查看、修改、操作其他 BU 的用户、数据或资源；BU 管理员所能分配的最高角色不超过自身权限上限（不可越权分配）。（来源：`05-01-系统功能-平台基础.md` FUNC-PLT-016；`04-领域对象-平台域.md` RBAC 约束）
- **校验逻辑**：BU 管理员发起操作时，系统校验目标资源的 tenantId 是否属于该管理员管理的 BU 子树；跨 BU 操作返回 403；角色分配时，系统校验被分配角色的权限范围是否超过 BU 管理员权限上限；超越则拒绝分配并记录审计。
- **异常处理**：跨 BU 操作尝试时，系统返回 403，提示"您无权操作其他 BU 的资源"，并记录 WARNING 级审计日志；越权角色分配时，提示"不可分配超过自身权限上限的角色"，操作被阻断。
- **配置项**：无
- **交互终端**：PC / Interface

### PLT-010 权限变更立即生效，并强制用户重新认证

- **等级**：MUST
- **触发场景**：管理员修改用户角色、撤销权限、停用账号时；用户所属 BU 被禁用时
- **规则描述**：用户权限发生变更（角色增减、权限撤销、账号停用、BU 禁用）时，变更须立即生效；系统将用户现有的所有 Token 加入黑名单，强制用户重新登录认证，不允许已下线的权限继续在 Token 有效期内被使用。（来源：`05-01-系统功能-平台基础.md` FUNC-PLT-004 / FUNC-PLT-023 约束）
- **校验逻辑**：权限变更操作保存后，系统立即将该用户所有 Access Token 和 Refresh Token 加入 Token 黑名单（有效期内不可使用）；黑名单检查在每次 API 请求的 Token 验证环节执行；黑名单缓存时间不超过 Token 有效期。
- **异常处理**：Token 黑名单服务不可用时，系统记录失败日志，告警运维人员，同时采用降级策略（缩短 Token 有效期为 5 分钟）确保权限变更最终生效；所有权限变更操作均记录审计日志，包含变更前后权限状态。
- **配置项**：Token 黑名单缓存有效期（默认等于 Token 有效期）；降级策略下的临时 Token 有效期（默认 5 分钟）
- **交互终端**：PC / Interface / 系统自动

### PLT-011 高危操作须记录 CRITICAL 级审计日志并触发告警

- **等级**：MUST
- **触发场景**：删除核心业务对象（数据集、模型、用户）；模型下发边端；批量权限变更；跨租户操作；导出敏感数据；修改全局系统配置时
- **规则描述**：系统预定义的高危操作列表中的每一项操作，无论执行结果如何，都必须生成 CRITICAL 风险等级的审计日志，并同步触发实时安全告警通知给安全管理员；仅记录 INFO 级日志不满足此规则。（来源：`04-领域对象-平台域.md` 高危操作分类定义）
- **校验逻辑**：业务逻辑层在执行高危操作前后，调用审计服务写入 CRITICAL 级日志（含操作前后状态的 before/after 字段）；审计服务同步触发告警通知（邮件 + IM）；操作失败（业务失败）的情况也须写入 CRITICAL 级日志（result = FAILURE）。
- **异常处理**：审计日志写入失败时，系统阻断对应的业务操作（写审计优先，业务操作为次）；告警发送失败时，记录告警失败日志，不阻断业务操作，但须在下一检查周期内重试告警。
- **配置项**：高危操作类型列表（可扩展，不可缩减）；告警通知渠道（邮件 / IM，多渠道同时通知）；告警接收人角色（默认安全管理员 + 超级管理员）
- **交互终端**：系统自动

### PLT-012 新用户创建后默认为零权限，所有权限须显式分配

- **等级**：MUST
- **触发场景**：超级管理员或 BU 管理员创建新用户时；SSO 首次登录自动创建账号时
- **规则描述**：新创建的用户账号默认不具备任何功能权限（无菜单访问、无资源操作权限）；所有权限必须通过显式的角色分配或权限授予操作获得，遵循最小权限原则；系统不允许为新用户预设"默认高权限角色"。（来源：`05-01-系统功能-平台基础.md` FUNC-PLT-027 约束：新用户创建后权限为零）
- **校验逻辑**：用户创建完成后，系统确保 `UserTenantRole` 表中无对应记录（零角色状态为新用户的初始合法状态，与 PLT-002 的约束时间点不同：PLT-002 约束的是已有用户被撤销为零角色的场景）；首次分配角色须由管理员显式操作并记录审计；SSO 自动创建的账号同样初始为零权限。
- **异常处理**：零权限用户登录时，系统允许登录成功但展示空白控制台，提示"您的账号暂无权限，请联系管理员为您分配角色"；所有权限分配操作均记录审计日志，包含分配人、被分配人、角色名称、生效时间。
- **配置项**：SSO 自动创建账号时是否分配默认最低权限角色（默认否，可按 BU 配置）；新用户角色分配完成后是否发送通知（默认是）
- **交互终端**：PC（管理员分配角色）/ 系统自动（账号创建）

### PLT-013 定期权限复审，高权限账号至少每季度审查一次

- **等级**：SHOULD
- **触发场景**：系统权限复审定时任务触发时；管理员发起临时权限复审时
- **规则描述**：系统应按配置的复审周期（默认每季度）自动提醒管理员对高权限账号（超级管理员、BU 管理员、数据所有者）进行权限复审，验证权限与当前岗位职责的匹配度，及时撤销冗余权限；复审结果须记录存档。（来源：`02-04-业务流程-平台运营.md` SYS-002 步骤 5；`05-01-系统功能-平台基础.md` FUNC-PLT-015）
- **校验逻辑**：复审定时任务在配置的周期到期后，自动生成复审工单分配给对应管理员；复审工单包含：用户权限清单、最近 90 天操作统计、权限与岗位匹配评估；管理员须在工单超时时限内完成复审并提交结论（保留 / 调整 / 撤销）；复审结果记录到审计日志。
- **异常处理**：复审超时未完成时，系统发送升级通知给上级管理员；不自动撤销权限（SHOULD 级，避免误操作）；但标记账号为"待复审"状态，后续权限申请须先完成复审。
- **配置项**：复审周期（BU 级可配置，默认季度）；高权限账号的判定标准（默认：SUPER_ADMIN + BU_ADMIN + 数据集所有者）；复审超时时限（默认 7 天）；是否自动撤销超期未复审账号的权限（默认否）
- **交互终端**：PC（管理员复审操作）/ 系统自动（提醒和工单创建）

### PLT-014 账号停用须触发强制下线并检查进行中任务

- **等级**：MUST
- **触发场景**：管理员执行用户停用（DISABLE）或注销（DEACTIVATE）操作时；用户离职流程触发时
- **规则描述**：账号停用操作触发后，系统须：①立即将该用户所有 Token 加入黑名单（强制下线）；②检查用户当前进行中的任务（标注任务、训练任务、标注审核）并发出告警；③撤销用户数据访问权限；以上步骤须在停用操作完成前全部执行。（来源：`05-01-系统功能-平台基础.md` FUNC-PLT-012 约束）
- **校验逻辑**：停用操作发起时，系统执行：①Token 黑名单写入（同步）；②扫描进行中任务（数据标注、模型训练、推理部署操作），生成告警清单展示给操作管理员；③撤销用户 ACL；③记录停用审计日志（含进行中任务清单）；用户注销（不可逆）时额外执行数据脱敏。
- **异常处理**：进行中任务无法自动迁移时，系统告警管理员手动处理（如将标注任务重新分配给其他标注员）；Token 黑名单写入失败时，阻断停用操作完成（不可接受因技术故障导致停用不彻底）；停用操作完成后向被停用用户的主管发送通知。
- **配置项**：停用前是否强制要求确认进行中任务处置方案（默认是）；账号停用后恢复期限（默认 90 天内可恢复，可配置）；注销是否需要双人审批（默认需要）
- **交互终端**：PC（管理员操作）/ Interface / 系统自动
