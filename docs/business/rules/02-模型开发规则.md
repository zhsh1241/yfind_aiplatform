# 02 模型开发规则

> **覆盖域**：模型开发与训练域（MODEL）——模型调优、训练任务、模型评估、模型管理、模型工程化、开发环境。  
> **规则总数**：13 条

---

### MDL-001 微调训练任务必须引用"已激活"状态的数据集

- **等级**：MUST
- **触发场景**：模型训练工程师提交微调 / 训练任务时
- **规则描述**：训练任务（包括微调、全量训练、分布式训练）引用的数据集必须处于"激活"（ACTIVE）状态；未激活（草稿、下线、归档、发布待审核）的数据集不得被训练任务挂载使用，防止因数据未就绪导致训练失败或产生不可复现结果。
- **校验逻辑**：任务提交时，系统对每个 `datasetId` 查询数据集状态；若存在任一数据集 status ≠ ACTIVE，则拒绝提交，返回具体数据集名称和当前状态。
- **异常处理**：校验失败时，系统提示"数据集 [名称] 状态为 [状态]，不可用于训练，请先激活数据集"；训练任务保持在"草稿"状态，不进入调度队列。
- **配置项**：无
- **交互终端**：PC / Interface

### MDL-002 训练任务提交前须通过 GPU/NPU 资源配额检查

- **等级**：MUST
- **触发场景**：模型训练工程师点击"提交训练任务"时，调度系统处理任务入队时
- **规则描述**：提交包含 GPU 或 NPU 资源申请的训练任务时，系统必须校验当前 BU 的剩余可用 GPU/NPU 配额；超过配额的任务不得立即进入调度队列（可排队等待，但不可抢占超出配额部分）。
- **校验逻辑**：系统查询该 BU 的 GPU/NPU 已分配量与总配额，计算剩余量；若本次申请量 > 剩余量，则拒绝即时调度；若 BU 配额管理策略为"排队"，则将任务加入等待队列，等待资源释放后按 FIFO 顺序调度。
- **异常处理**：即时提交失败时，系统提示"当前 BU GPU 配额不足（已用 X / 总量 Y），任务已加入等待队列"；排队任务在资源就绪时自动触发调度，并通过消息通知用户。若 BU 管理员未申请过该资源类型，则提示"请先申请资源配额"。
- **配置项**：BU 级 GPU/NPU 配额总量（由超级管理员配置）；配额超限行为（拒绝 / 排队，默认排队）；排队超时告警时限
- **交互终端**：PC / Interface / 系统自动

### MDL-003 存在活跃推理部署的模型版本不得删除

- **等级**：MUST
- **触发场景**：模型训练工程师或模型管理员执行模型版本删除操作时
- **规则描述**：若模型版本当前存在处于"运行中"或"启用"状态的推理服务部署，该模型版本不允许被删除，防止推理服务因模型文件缺失而中断。
- **校验逻辑**：删除操作触发时，系统查询推理服务部署表，检查是否存在 `modelVersionId = {targetVersionId}` 且 `deploymentStatus IN (RUNNING, ENABLED)` 的记录；存在则阻断删除。
- **异常处理**：校验失败时，系统提示"该模型版本当前被以下推理服务引用（列表），请先下线相关服务后再删除"；操作被阻断；引导用户先下线对应推理服务。
- **配置项**：无
- **交互终端**：PC / Interface

### MDL-004 跨 BU 模型共享须经 BU 管理员审批

- **等级**：MUST
- **触发场景**：用户将 BU 级或项目级模型的访问范围变更为跨 BU 可见时；其他 BU 用户申请访问非本 BU 模型时
- **规则描述**：团队模型（BU 级或项目级 scope）的跨 BU 共享，须经模型 owner 授权（owner 为创建该模型的工程师或其所在 BU 管理员）；未经授权，其他 BU 的用户不得访问、引用或基于该模型进行训练。模型查看/下载/推理服务均需模型 owner 授权（BIZ-01 已确认）。
- **校验逻辑**：访问非本 BU 模型时，系统检查是否存在有效的跨 BU 授权记录（授权状态为"已通过"且未过期）；若无，返回 403；模型 scope 变更为跨 BU 时，系统触发审批工作流，审批通过前变更不生效。
- **异常处理**：无授权访问时返回 403，提示"该模型属于其他 BU，请申请跨 BU 授权"；审批被拒绝时，scope 保持原值，申请人收到拒绝通知和原因。
- **配置项**：跨 BU 模型授权有效期（默认永久，可配置）；审批级别（单级 BU 管理员 / 双级，待确认：BIZ-01）
- **交互终端**：PC（申请 / 审批）/ Interface

### MDL-005 AutoML 超参数搜索空间必须设定上界

- **等级**：MUST
- **触发场景**：模型训练工程师配置自动超参数调优（AutoML）任务，定义搜索空间时
- **规则描述**：AutoML 超参数搜索空间的每个数值型参数必须设定有限的上界（max 值），不允许配置无上界的连续搜索范围，防止无限计算资源消耗和任务长期不收敛。
- **校验逻辑**：AutoML 任务提交时，系统遍历搜索空间定义，对每个数值型参数（学习率、批次大小等）校验是否存在有效的 max 值（非 null、非 Infinity、大于 min 值）；不满足则拒绝提交。
- **异常处理**：校验失败时，系统提示"超参数 [参数名] 未设置上界，请完善搜索空间定义"；任务提交被阻断。
- **配置项**：各超参数类型的系统默认上界推荐值（如学习率上界建议 0.1）
- **交互终端**：PC / Interface

### MDL-006 模型必须通过评估后方可发布到模型注册中心

- **等级**：MUST
- **触发场景**：模型训练工程师或模型管理员执行模型版本"发布到模型注册中心"操作时
- **规则描述**：模型版本发布到模型注册中心（平台级或 BU 级可用）前，必须存在一条针对该版本的"已通过"评估记录；评估未通过或从未经过评估的模型版本不允许发布，防止低质量模型进入生产使用链路。
- **校验逻辑**：发布操作触发时，系统查询模型评估记录表，检查该 `modelVersionId` 是否存在 `evaluationStatus = PASSED` 的评估记录；若不存在则阻断发布。
- **异常处理**：校验失败时，系统提示"该模型版本尚未通过评估，请先执行模型评估并达到通过标准后再发布"；引导用户跳转到模型评估页面。
- **配置项**：评估通过标准（由任务配置时定义，如准确率 ≥ 阈值）；是否强制要求独立验证集（不可使用训练集，建议配置为强制）
- **交互终端**：PC / Interface

### MDL-007 同一用户在同一项目内只允许存在一个活跃开发环境

- **等级**：MUST
- **触发场景**：模型训练工程师创建新的开发环境（Notebook / VSCode / SSH）时
- **规则描述**：同一用户在同一项目（projectId）下，同时只允许运行一个活跃的开发环境实例；重复创建时系统阻断，防止资源无效占用。
- **校验逻辑**：创建开发环境时，系统查询该 `userId + projectId` 组合下状态为"运行中"的开发环境数量；若数量 ≥ 1 则拒绝创建新实例。
- **异常处理**：校验失败时，系统提示"您在该项目下已有一个活跃的开发环境（[实例名称]），请先停止或删除现有环境后再创建新的"；引导用户管理现有实例。
- **配置项**：单用户单项目最大并发开发环境数量（默认 1，超级管理员可调整）
- **交互终端**：PC / Interface

### MDL-008 训练任务使用的基础镜像必须通过安全扫描

- **等级**：MUST
- **触发场景**：模型训练工程师或开发工程师在训练任务或开发环境中选择 Docker 镜像时
- **规则描述**：训练任务和开发环境使用的基础镜像必须已通过安全扫描（无高危漏洞）；未经安全扫描或存在高危漏洞的镜像不允许被选择使用（参见 RES-004）。
- **校验逻辑**：镜像选择列表仅展示 `securityScanStatus = PASSED` 的镜像；通过 API 传入镜像 ID 时，系统校验镜像安全扫描状态；未通过扫描则拒绝任务创建。
- **异常处理**：校验失败时，系统提示"所选镜像未通过安全扫描，无法使用，请选择其他已扫描镜像或联系镜像管理员"。
- **配置项**：高危漏洞拦截级别（Critical / High，默认 Critical 必须拦截）；扫描工具配置（Trivy / Clair 等）
- **交互终端**：PC / Interface / 系统自动

### MDL-009 模型版本状态机须按既定顺序流转，不得跳跃

- **等级**：MUST
- **触发场景**：模型管理员或训练工程师变更模型版本生命周期状态时
- **规则描述**：模型版本状态按 开发（Development）→ 测试（Testing）→ 生产（Production）→ 废弃（Deprecated）的顺序流转；不允许从"开发"直接跳转到"生产"，必须经过"测试"阶段；状态不可逆向流转（如不可从"生产"降回"测试"）。
- **校验逻辑**：状态变更操作触发时，系统检查目标状态是否为当前状态的合法后继状态；合法转换矩阵：Development → Testing、Testing → Production 或 Deprecated、Production → Deprecated；其余转换均被拒绝。
- **异常处理**：非法状态转换时，系统提示"不支持从 [当前状态] 直接转换为 [目标状态]，请按状态流转顺序操作"；操作被阻断。
- **配置项**：无
- **交互终端**：PC / Interface

### MDL-010 分布式训练任务最少需要 2 个节点

- **等级**：MUST
- **触发场景**：模型训练工程师配置分布式训练任务，设置节点数量时
- **规则描述**：提交分布式训练任务时，指定的节点数量（worker 数量）必须 ≥ 2；节点数为 1 的配置本质上是单机任务，不允许以分布式训练任务类型提交，防止分布式框架开销浪费。
- **校验逻辑**：分布式训练任务提交时，系统校验 `workerCount` 字段；若 `workerCount < 2` 则拒绝提交，提示改用单机训练任务类型。
- **异常处理**：校验失败时，系统提示"分布式训练任务至少需要 2 个节点，当前配置为 1 个节点，请选择单机训练任务类型或增加节点数量"；操作被阻断。
- **配置项**：分布式训练最大节点数上限（按资源池配置）
- **交互终端**：PC / Interface

### MDL-011 模型工程化优化后须验证精度损失在可接受范围内

- **等级**：SHOULD
- **触发场景**：模型工程化（量化 / 剪枝 / 蒸馏）任务完成，生成优化模型后
- **规则描述**：模型工程化完成后，系统应自动执行精度对比验证，计算优化前后模型在标准验证集上的精度差值；精度损失超过阈值时，应向用户发出警告，提示风险，建议调整优化策略后重新执行。
- **校验逻辑**：工程化任务完成后，系统触发精度对比任务：使用同一验证集对原始模型和优化后模型各执行一次推理评估；计算主要指标差值，若 `精度差值 > 配置阈值`，则标记为"精度警告"。
- **异常处理**：超出阈值时，系统显示警告"优化后精度损失 [X]% 超过允许阈值 [Y]%，请调整优化参数或选择其他优化策略"；优化结果仍可保存，但带警告标记，不阻断工作流（SHOULD 级）。
- **配置项**：可接受的最大精度损失比例（默认 2%，可按任务配置）；对比验证集 ID（可指定或使用原训练测试集）
- **交互终端**：系统自动 / PC

### MDL-012 训练任务支持断点续训，中断后可从最近检查点恢复

- **等级**：SHOULD
- **触发场景**：训练任务因资源抢占、节点故障或手动停止而中断时；用户重新提交被中断的训练任务时
- **规则描述**：训练任务应支持从最近保存的检查点（checkpoint）恢复训练，避免因意外中断导致全部训练进度丢失；系统应在任务配置中提供检查点保存间隔设置，并在重启时自动检测可用检查点。
- **校验逻辑**：任务重启时，系统扫描该任务的检查点目录，若存在有效检查点文件，则在提交前询问用户是否从检查点继续；若用户确认，系统在启动命令中自动注入检查点路径参数。
- **异常处理**：检查点文件损坏时，系统告警并提示用户选择更早的检查点或从头重新训练；无检查点时，系统提示从第 0 步开始训练。
- **配置项**：检查点自动保存间隔（默认每 100 步，可配置）；检查点最大保留数量（默认保留最近 3 个）
- **交互终端**：PC / Interface / 系统自动

### MDL-013 模型工程化格式转换失败须记录日志并支持重试

- **等级**：MUST
- **触发场景**：模型工程化任务（量化、剪枝、蒸馏、格式转换等）执行失败时；用户主动重试工程化任务时
- **规则描述**：模型工程化任务（包括格式转换、量化压缩、剪枝、蒸馏等）失败时，系统必须完整记录失败日志（含错误原因、堆栈信息、输入参数），并支持用户在不重新上传模型的情况下原地重试；重试最多允许 3 次，超过次数后任务进入"失败（FAILED）"终态，需人工干预。
- **校验逻辑**：任务进入失败状态时，系统自动将错误日志写入任务详情页；允许用户针对同一任务配置触发重试（保持相同参数或修改参数后重试）；重试计数器 `retryCount` 每次自增，达到 `maxRetry`（默认 3）时拒绝继续重试。
- **异常处理**：格式不兼容导致失败时，系统在日志中提示"模型格式与目标框架不兼容，请检查模型导出格式"；内存不足导致失败时，系统提示"资源不足，建议调整量化参数或申请更大资源配额"；超过最大重试次数时，系统提示"已达最大重试次数（3 次），请联系平台运维"。
- **配置项**：最大重试次数（默认 3 次，可配置）；失败日志保留时长（默认 30 天，可配置）
- **交互终端**：PC / Interface / 系统自动
