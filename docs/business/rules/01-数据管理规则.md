# 01 数据管理规则

> **覆盖域**：数据管理域（DATA）——数据源接入、多模态采集与加工、数据标注、数据集生命周期、数据增强、数据集门户。  
> **规则总数**：12 条

---

### DAT-001 数据源连接测试必须通过方可激活

- **等级**：MUST
- **触发场景**：数据标注工程师或 BU 管理员完成数据源配置参数填写，点击"保存激活"时
- **规则描述**：数据源在激活（纳管）前，系统必须执行连接可用性测试；测试未通过的数据源不允许进入激活状态，不得被后续采集任务引用。
- **校验逻辑**：保存激活操作触发后，系统向目标数据源发起探测请求（TCP 握手 + 协议握手），在配置超时时限内收到正常响应则判定通过；超时或收到错误响应则判定失败。
- **异常处理**：连接测试失败时，系统拒绝激活操作，将数据源状态保持在"未激活"，页面展示具体失败原因（如连接超时、认证失败、网络不可达），引导用户修正配置参数或检查网络连通性后重试。
- **配置项**：连接测试超时时限（默认 30 秒，可配置）；重试次数（默认 1 次）
- **交互终端**：PC / Interface

### DAT-002 内容安全检测为数据入平台的强制前置步骤

- **等级**：MUST
- **触发场景**：多模态数据采集完成，数据即将进入加工流水线时；用户手动上传数据集时
- **规则描述**：所有进入平台的数据（文本、图片、音频、视频）在正式纳入数据集前，必须经过内容安全检测，识别涉黄、涉暴、涉恐、违法违规等敏感内容；未通过内容安全检测的数据不得进入任何后续流程。
- **校验逻辑**：系统调用内容安全检测服务，对批次内每条数据进行分级检测（安全 / 低风险 / 中风险 / 高风险）；高风险内容自动拦截；中风险内容进入人工审核队列；仅安全和低风险内容允许继续进入加工流水线。
- **异常处理**：检测到高风险内容时，系统自动拦截该批次中的违规数据条目，标记为"内容安全拦截"，通知管理员处置；整批次其余合规数据可继续处理。检测服务不可用时，系统暂停采集任务，告警运维人员。
- **配置项**：内容安全检测服务接入配置（对接第三方内容安全 API，检测政治、暴恐、色情等违规内容，BIZ-04 已确认）；风险等级阈值；中风险内容是否需要人工审核开关
- **交互终端**：系统自动 / PC（内容审核员处理中风险队列）

### DAT-003 标注任务必须配置标签模板方可分配执行

- **等级**：MUST
- **触发场景**：标注任务管理员创建标注任务，尝试分配标注人员并启动任务时
- **规则描述**：标注任务在进入执行状态前，必须完成标签模板（标签体系、标注规范）的配置；未配置标签模板的任务不允许分配给标注工程师，不允许启动标注工作。（注：标签模板不包含质量标准，质量评审由数据审核工程师基于业务场景经验判断。）
- **校验逻辑**：任务状态流转至"分配中"或"执行中"时，系统检查标注任务是否存在有效的标签模板记录（模板状态为已发布）；若缺失则阻断状态流转。
- **异常处理**：校验失败时，系统提示"任务尚未配置标签模板，请先完成标签体系定义"，阻止任务分配操作；标注任务保持在"草稿"状态。
- **配置项**：无
- **交互终端**：PC（标注任务管理员）

### DAT-004 标注工程师不得审核自己提交的标注结果

- **等级**：MUST
- **触发场景**：审核工程师在标注审核页面选择待审核标注记录时；系统自动分配审核任务时
- **规则描述**：为保证标注质量的客观性，同一标注记录的提交人与审核人不得为同一用户；标注工程师提交的标注结果，只能由其他用户（审核工程师角色）进行审核，不允许自我审核。
- **校验逻辑**：审核操作发起时，系统比对审核操作人的 userId 与该标注记录的 annotatorId；若两者相同，则拒绝审核操作。自动分配审核任务时，系统过滤掉与标注人相同的用户。
- **异常处理**：校验失败时，系统提示"不允许审核自己提交的标注结果"，操作被阻断；系统自动将该任务分配给其他有效的审核工程师。
- **配置项**：无
- **交互终端**：PC（审核工程师）/ 系统自动

### DAT-005 已发布的数据集版本不得修改，变更须新建版本

- **等级**：MUST
- **触发场景**：用户尝试编辑已处于"已发布"状态的数据集版本内容时；用户尝试删除已发布版本的数据条目时
- **规则描述**：数据集版本一旦发布（状态变更为"已发布/激活"），其数据内容、元数据结构不得被修改或删除，以保证训练任务的可复现性和数据血缘的完整性。需要修改时，必须新建版本号。
- **校验逻辑**：对数据集版本的写操作（UPDATE / DELETE）执行前，系统检查版本状态；若状态为"已发布"或"激活"，则拒绝写操作。
- **异常处理**：校验失败时，系统提示"已发布版本不可修改，如需变更请新建版本"，操作被阻断；引导用户使用"新建版本"功能进行迭代。
- **配置项**：无
- **交互终端**：PC / Interface

### DAT-006 受限数据集访问须经审批，授权具有有效期

- **等级**：MUST
- **触发场景**：用户在数据集门户申请访问"受限（Restricted）"级别数据集时；受限数据集访问权限到期时
- **规则描述**：访问受限级别数据集必须提交申请并经数据集所有者或 BU 管理员审批；审批通过后授予的访问权限必须设置有效期，期满后权限自动撤销；公开数据集无需此流程。
- **校验逻辑**：用户发起受限数据集访问请求时，系统检查是否存在有效审批记录（审批状态为"已通过"且有效期未过期）；若无有效记录则拒绝访问，返回 403。权限有效期由定时任务检查，到期后自动撤销 ACL。
- **异常处理**：无有效权限时，系统提示"该数据集为受限访问，请提交申请"，引导用户进入申请流程；申请被拒绝时展示拒绝原因；权限到期自动撤销并发送通知提醒用户重新申请。
- **配置项**：默认访问授权有效期（可配置，建议 30 天）；审批超时自动拒绝时限（默认 48 小时，可配置；超时后系统自动将申请状态置为"已拒绝"并通知申请人）
- **交互终端**：PC（申请 / 审批）/ 系统自动（权限到期撤销）

### DAT-007 数据增强数据集必须关联源数据集，血缘不可缺失

- **等级**：MUST
- **触发场景**：数据增强任务完成，系统生成增强数据集时；用户尝试手动创建增强数据集时
- **规则描述**：所有通过数据增强流程生成的数据集，必须在元数据中记录源数据集 ID 和增强策略参数，建立完整的数据血缘关系；血缘信息缺失的增强数据集不得发布。
- **校验逻辑**：增强数据集发布前，系统校验其元数据中 `sourceDatasetId` 字段是否非空，且对应的源数据集在平台中存在；同时校验增强策略参数（类型、配置）是否已记录；任一缺失则阻止发布。
- **异常处理**：校验失败时，系统提示"增强数据集缺少血缘信息，无法发布"；系统记录增强任务日志，运维人员可通过日志修复血缘信息。
- **配置项**：无
- **交互终端**：系统自动（增强任务完成时）/ PC（人工创建时）

### DAT-008 被安全检测标记的敏感内容须由管理员处置，禁止自动删除

- **等级**：MUST
- **触发场景**：内容安全检测将数据条目标记为"高风险"或"待确认"后，触发后续处置流程时
- **规则描述**：被内容安全检测标记为敏感的数据内容，不得由系统自动删除；必须进入人工处置队列，由具备内容审核权限的管理员确认处置方式（隔离 / 脱敏 / 删除 / 误报解除），以确保合规性和处置可追溯。
- **校验逻辑**：内容安全服务输出高风险结果后，系统将数据状态置为"待管理员处置"；任何针对该数据的自动删除操作在系统层面被禁止，删除操作仅接受来自管理员角色的显式指令。
- **异常处理**：若系统检测到非管理员角色触发的敏感内容删除请求，操作被拒绝，记录 CRITICAL 级审计日志，并通知安全管理员。
- **配置项**：内容审核管理员角色定义；处置超时告警时限（默认 48 小时未处置则升级告警）
- **交互终端**：PC（内容审核管理员）/ 系统自动（标记与队列管理）

### DAT-009 标注任务须使用已激活状态的数据集（原始数据集或预处理数据集均可）

- **等级**：MUST
- **触发场景**：标注任务管理员创建标注任务，选择待标注数据集时
- **规则描述**：标注任务引用的数据集（原始数据集或预处理数据集均可）必须处于"激活"状态（已发布且可用）；处于草稿、待发布、已归档、已下线等非激活状态的数据集不得被标注任务引用，防止标注结果与不完整数据关联。
- **校验逻辑**：创建标注任务时，系统过滤数据集选择列表，仅展示 status = ACTIVE 的数据集；若通过 API 传入非激活数据集 ID，系统在任务创建校验阶段拒绝，返回错误信息。
- **异常处理**：校验失败时，系统提示"所选数据集状态不可用，请先激活数据集"，阻断任务创建；引导用户激活对应数据集。
- **配置项**：无
- **交互终端**：PC / Interface

### DAT-010 标注数据集发布前须通过质量检查

- **等级**：MUST
- **触发场景**：标注任务管理员或数据管理员执行标注数据集"发布激活"操作时
- **规则描述**：标注数据集在发布前，必须通过系统质量检查（数据完整性验证、标注结果格式校验、覆盖率检查）；质量检查未通过的标注数据集不得发布，不得被模型训练引用。
- **校验逻辑**：发布操作触发系统执行：①数据条目完整性检查（无缺失 label）；②格式合规性校验（符合标注场景定义的 schema）；③标注覆盖率计算（已标注条目 / 总条目 ≥ 配置阈值）；三项全部通过方可发布。
- **异常处理**：质量检查失败时，系统展示具体失败项（如"覆盖率 78% 未达到阈值 90%"），阻断发布；引导任务管理员补充标注后重新提交质量检查。
- **配置项**：标注覆盖率最低阈值（默认 90%，可按 BU 配置）；格式校验规则（按标注场景配置）；标注审核通过基于场景业务经验判断，无统一 Cohen's Kappa 等量化标准（BIZ-02 已确认）
- **交互终端**：PC（标注任务管理员）/ 系统自动

### DAT-011 数据集删除前须检查是否存在关联的训练任务引用

- **等级**：MUST
- **触发场景**：数据管理员或数据所有者执行数据集删除操作时
- **规则描述**：数据集（任意类型：原始 / 预处理 / 标注 / 增强）在删除前，系统必须检查是否存在引用该数据集的活跃训练任务或已发布的模型版本；存在引用时不允许删除，防止训练任务因数据缺失而失败。
- **校验逻辑**：删除操作发起时，系统查询关联关系表，检查：①是否有状态为"运行中"或"排队中"的训练任务引用该数据集；②是否有模型版本的 lineage 记录关联该数据集；任一存在则阻断删除。
- **异常处理**：校验失败时，系统提示"该数据集被以下训练任务/模型版本引用，不可删除"，展示引用清单；用户须先解除引用关系或等待任务完成后再操作。数据集回收后可物理删除，但须严格管控删除权限（仅超级管理员可执行，BIZ-03 已确认）。
- **配置项**：无
- **交互终端**：PC / Interface

### DAT-012 数据集访问遵循 BU 数据隔离，跨 BU 访问须授权

- **等级**：MUST
- **触发场景**：用户访问、搜索、下载数据集时；API 请求数据集资源时
- **规则描述**：数据集的可见性和访问权限默认限定在创建该数据集的 BU（tenantId）范围内；跨 BU 访问须经数据集所属 BU 管理员的显式授权（参见 FUNC-PLT-037）；系统所有数据集查询自动附加 tenantId 过滤条件。
- **校验逻辑**：所有数据集查询请求经租户上下文注入后，系统在 SQL/查询层自动附加 `WHERE tenant_id = {currentTenantId}` 过滤；跨 BU 访问请求检查跨 BU 授权记录是否存在且有效；无授权记录时返回 404（不暴露资源存在性）。
- **异常处理**：跨 BU 无授权访问时，返回 404；授权记录过期时，权限自动撤销，访问返回 403 并提示"访问权限已过期，请重新申请"。
- **配置项**：跨 BU 数据集授权有效期（默认 30 天，可配置）
- **交互终端**：PC / Interface / 系统自动
