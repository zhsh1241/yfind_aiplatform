# 03 推理部署规则

> **覆盖域**：推理域（INF）——中心端推理服务部署、第三方推理服务纳管、边端服务器模型下发、批量推理、流量控制与回滚。  
> **规则总数**：11 条

---

### INF-001 仅已发布/已审批的模型版本可部署推理服务

- **等级**：MUST
- **触发场景**：模型工程师或运维工程师选择模型版本，触发中心端推理服务部署流程时
- **规则描述**：推理服务部署只允许使用状态为"生产（Production）"的模型版本；处于"开发（Development）"或"测试（Testing）"状态的模型版本不得部署到生产推理服务，防止未经充分验证的模型对外提供服务。
- **校验逻辑**：部署操作发起时，系统查询目标 `modelVersionId` 的状态；若 `status ≠ Production`，则拒绝部署请求，返回错误信息及当前模型状态。
- **异常处理**：校验失败时，系统提示"所选模型版本当前处于 [状态] 阶段，仅生产状态的模型版本可部署推理服务，请先完成模型评估和状态流转"；部署操作被阻断。
- **配置项**：无
- **交互终端**：PC / Interface

### INF-002 多版本流量分配比例之和必须等于 100%

- **等级**：MUST
- **触发场景**：运维工程师或推理服务管理员配置 A/B 测试或灰度发布的流量分配策略时；修改现有多版本推理服务的流量比例时
- **规则描述**：同一推理服务下部署多个模型版本时，各版本的流量分配比例之和必须精确等于 100%；不允许提交总比例不等于 100% 的流量配置，防止请求丢失或未定义路由行为。
- **校验逻辑**：保存流量配置时，系统对所有活跃版本的 `trafficRatio` 求和；若 `sum ≠ 100`（考虑浮点精度，允许 ±0.01% 的误差），则拒绝保存，返回当前总量。
- **异常处理**：校验失败时，系统提示"流量比例之和为 [X]%，必须等于 100%，请调整各版本分配"；配置保持原值，不生效。
- **配置项**：无
- **交互终端**：PC / Interface

---

### INF-003 边端模型下发须经审批流程后方可执行

- **等级**：MUST
- **触发场景**：运维工程师触发"立即下发"或"定时下发"边端模型推送任务时
- **规则描述**：模型下发到边端生产服务器须经边端应用 owner 授权；未经授权的下发任务不得执行，防止未经授权的模型变更影响边端生产业务（BIZ-05 已确认）。
- **校验逻辑**：下发任务提交时，系统检查是否存在针对该次下发操作（边端服务器 ID + 模型版本 ID + 时间戳）的审批记录，且审批状态为"已通过"；若无有效审批记录，则拒绝下发任务执行。
- **异常处理**：无审批记录时，系统自动创建审批工单，通知有权限的审批人（待确认：BIZ-05），并提示操作人"下发申请已提交，等待审批通过后执行"；审批被拒绝时，下发任务取消，操作人收到拒绝通知。
- **配置项**：审批人角色配置（默认为 BU 管理员，待确认：BIZ-05）；审批超时自动拒绝时限（默认 24 小时）；是否需要二级审批（待确认：BIZ-05）
- **交互终端**：PC（审批）/ Interface / 系统自动

### INF-004 批量推理任务启动前须验证输入数据集有效性

- **等级**：MUST
- **触发场景**：用户提交批量离线推理任务时
- **规则描述**：批量推理任务启动前，系统必须对输入数据集进行有效性验证，包括：数据集存在性、状态为激活、格式与推理服务 schema 兼容、数据量非零；任一验证失败则不允许启动批量推理任务。
- **校验逻辑**：任务提交时，系统依次执行：①查询数据集是否存在且 status = ACTIVE；②检查数据集格式是否与推理服务声明的输入 schema 兼容；③统计数据集有效条目数，若为 0 则拒绝；全部通过后任务进入调度队列。
- **异常处理**：校验失败时，系统提示具体失败原因（如"数据集格式不兼容：推理服务期望 JSON 格式，实际为 CSV"）；批量任务不进入队列，保持草稿状态。
- **配置项**：格式兼容性校验规则（按推理服务 schema 配置）
- **交互终端**：PC / Interface

### INF-005 生产推理服务最小实例数不得低于 1

- **等级**：MUST
- **触发场景**：运维工程师配置推理服务扩缩容策略时；扩缩容策略触发自动缩容操作时
- **规则描述**：标记为"生产（Production）"环境的推理服务，其最小副本数（minReplicas）的配置值不得低于 1；防止生产服务在低流量期间缩容至 0 导致服务不可用（冷启动延迟）。
- **校验逻辑**：保存扩缩容配置时，若服务环境标记为"生产"且 `minReplicas < 1`，系统拒绝保存；自动缩容策略执行时，调度系统在下调副本数前检查下调后是否低于 minReplicas 配置值，低于则停止缩容。
- **异常处理**：配置保存时校验失败，提示"生产环境推理服务最小实例数不得低于 1，当前设置为 0"；自动缩容触达最小值时，系统停止缩容并记录日志，不触发告警（正常行为）。
- **配置项**：生产服务最小实例数下限（默认 1，超级管理员可调整）；开发/测试环境是否允许缩容至 0（默认允许，节省资源）
- **交互终端**：PC / Interface / 系统自动

### INF-006 新版本健康检查失败时自动回滚至上一版本

- **等级**：MUST
- **触发场景**：推理服务完成新版本部署，拨测验证阶段失败时；新版本上线后健康检查持续失败时
- **规则描述**：推理服务新版本部署后，在拨测验证期间或上线后的健康检查监控期（默认 10 分钟）内，若健康检查连续失败达到阈值，系统必须自动回滚到上一个稳定运行的版本，并触发告警通知。
- **校验逻辑**：系统在新版本上线后启动健康检查监控，在窗口期内检测 HTTP 健康端点响应；若连续失败次数 ≥ 配置阈值（默认 3 次），则触发自动回滚：将流量切换回上一版本，并将新版本标记为"部署失败"。
- **异常处理**：回滚触发后，系统向服务负责人发送告警通知，包含失败详情（错误码、响应时间、日志摘要）；回滚完成后记录 CRITICAL 级审计日志；若上一版本也不可用，则告警升级，通知运维团队人工介入。
- **配置项**：健康检查连续失败触发回滚阈值（默认 3 次）；健康检查间隔（默认 30 秒）；新版本监控窗口期（默认 10 分钟）；回滚通知渠道（邮件 / IM）
- **交互终端**：系统自动

### INF-007 推理服务下线前须通知调用方并设置冷却期

- **等级**：SHOULD
- **触发场景**：运维工程师或推理服务管理员执行推理服务"下线"操作时
- **规则描述**：推理服务正式下线前，系统应向已注册的调用方（API Key 持有者 / 订阅用户）发送下线通知，并保留不少于配置时长的冷却期（服务继续响应但标记为即将下线）；直接强制下线不发通知的操作仅允许在紧急安全事件时使用。
- **校验逻辑**：常规下线操作发起时，系统检查是否已发送过下线通知且冷却期已到；若未满足，则提示执行预通知流程；允许用户选择"紧急下线（跳过冷却期）"，但需要填写原因并记录审计。
- **异常处理**：下线通知发送失败（如邮件服务不可用）时，系统记录失败日志并告警，建议运维人员手动通知；不阻断下线操作本身。
- **配置项**：下线冷却期时长（默认 7 天，可配置）；调用方通知渠道（邮件 / 应用内通知）；紧急下线是否需要二次确认（默认需要）
- **交互终端**：PC / Interface / 系统自动

### INF-008 边端服务器模型下发须验证文件完整性

- **等级**：MUST
- **触发场景**：模型文件推送到边端服务器完成后，验证阶段执行时
- **规则描述**：模型文件下发到边端服务器后，系统必须自动验证文件完整性（校验和比对），确保传输过程中文件未损坏或篡改；完整性校验失败的下发任务不得启动边端推理服务，须重新下发。
- **校验逻辑**：下发任务在文件传输完成后，自动计算边端接收文件的 SHA-256 校验和，与平台侧记录的原始文件校验和进行比对；一致则继续部署，不一致则标记下发失败，触发重传。
- **异常处理**：完整性校验失败时，系统标记该次下发为"失败"，自动触发重传（最多重试 3 次）；重试 3 次仍失败后，告警通知运维人员和边端维护人员排查网络或存储问题。
- **配置项**：文件完整性校验算法（默认 SHA-256）；自动重传最大次数（默认 3 次）；重传间隔（默认 5 分钟）
- **交互终端**：系统自动

### INF-009 推理服务配置变更须记录变更历史，关键变更需二次确认

- **等级**：SHOULD
- **触发场景**：运维工程师修改推理服务的资源配额、流量限制、安全围栏规则、扩缩容策略等关键参数时
- **规则描述**：推理服务的关键配置变更（资源配额调整、QPS 限制修改、安全策略变更、流量切换）应记录完整的变更历史（before / after），并对高影响变更要求操作人二次确认，防止误操作。
- **校验逻辑**：系统对预定义的关键配置字段（资源配额、QPS 上限、安全过滤开关）的变更触发二次确认对话框，展示变更前后对比；操作人确认后方可保存；所有保存操作自动记录到审计日志（含 before/after 字段）。
- **异常处理**：操作人取消二次确认时，配置保持原值，记录"操作取消"审计条目；二次确认无响应超时（5 分钟）时，操作自动取消。
- **配置项**：需要二次确认的关键配置字段列表（可配置）；二次确认超时时限（默认 5 分钟）
- **交互终端**：PC / Interface

### INF-010 第三方推理服务纳管须通过连接测试且配置访问有效期

- **等级**：MUST
- **触发场景**：运维工程师或推理服务管理员将第三方 AI 推理服务纳管到平台时
- **规则描述**：第三方推理服务在纳管激活前，必须通过连接测试（URL 可达 + 认证信息有效）；同时必须配置服务可用期限，不允许纳管无期限的第三方服务，防止长期使用的服务失效后无人感知。
- **校验逻辑**：纳管操作触发连接测试，向第三方服务发送测试请求；测试通过且 `availableUntil` 字段非空时方可激活；`availableUntil` 为必填字段，系统在保存时校验其非空且为未来时间。
- **异常处理**：连接测试失败时，系统展示失败原因并阻断激活；`availableUntil` 缺失时，提示"请配置服务可用期限"；服务到期前 7 天系统发送到期提醒通知；到期后服务自动停用，使用方收到告警。
- **配置项**：连接测试超时时限（默认 10 秒）；到期提醒提前天数（默认 7 天）；到期后自动停用开关（默认开启）
- **交互终端**：PC / Interface / 系统自动

### INF-011 边端服务器离线期间日志须本地缓存，恢复连接后自动同步

- **等级**：MUST
- **触发场景**：边端服务器与中心端网络断开，推理任务仍在边端本地执行时；边端服务器网络恢复，重新与中心端建立连接时
- **规则描述**：边端服务器在离线状态下执行的推理任务日志、告警事件、健康检查记录必须在本地持久化缓存，不得丢弃；网络恢复后，边端 Agent 必须自动将缓冲日志按时序上传至中心端，且上传过程须幂等（允许重复上传同一批次日志，中心端去重处理）；离线时长超过配置阈值时，中心端应标记该边端为"长期离线"状态并触发告警。
- **校验逻辑**：边端 Agent 检测到中心端连接断开后，将后续产生的日志写入本地缓冲区（磁盘，非内存）；连接恢复后，Agent 扫描缓冲区中未同步的日志批次，按时间顺序逐批上传并在成功响应后清除本地副本；中心端接收日志时以 `(edgeServerId, logBatchId)` 为唯一键去重。
- **异常处理**：本地缓冲区超过磁盘配额时，丢弃最早批次并在恢复后于日志中标注"部分日志因磁盘空间不足已丢弃，丢弃时间范围 [start, end]"；同步失败时按指数退避重试（初始 30 秒，最大 30 分钟）；中心端离线时长超阈值告警触发后，需人工确认方可解除告警。
- **配置项**：本地日志缓冲区磁盘配额（默认 2 GB，可配置）；离线超阈值告警时限（默认 24 小时，可配置）；同步重试最大间隔（默认 30 分钟，可配置）
- **交互终端**：边端自动（日志缓存与同步）/ PC（告警确认与运维）
