# 04 资源管理规则

> **覆盖域**：资源域（RES）——计算资源配额、GPU/NPU 调度、存储管理、镜像管理、开发环境生命周期。  
> **规则总数**：10 条

---

### RES-001 BU 不得超过分配的计算资源配额，超额请求直接拒绝

- **等级**：MUST
- **触发场景**：用户提交训练任务、创建开发环境、部署推理服务时申请计算资源；BU 管理员提交资源使用申请时
- **规则描述**：每个 BU 拥有超级管理员分配的计算资源配额（CPU、内存、GPU/NPU 总量）；BU 内所有任务的资源申请总量不得超过该 BU 的配额上限；超过配额的资源申请直接拒绝（或进入等待队列，视配置而定）。
- **校验逻辑**：资源申请发起时，系统查询该 BU 当前已占用资源量（运行中任务 + 排队中任务的申请量）与配额上限；若 `已占用 + 本次申请 > 配额上限`，则拒绝立即分配；等待队列模式下任务入队但不立即分配资源。
- **异常处理**：超额时系统提示"BU [名称] 当前资源使用已达配额上限（已用 X，总额 Y），请等待资源释放或由 BU 管理员向总部申请扩容"；任务进入等待队列，资源释放后按 FIFO 顺序自动调度。
- **配置项**：BU 级 CPU/内存/GPU/NPU 配额总量（超级管理员配置）；超额行为（拒绝 / 排队，默认排队）；等待队列最大长度（默认 50）
- **交互终端**：PC / Interface / 系统自动

### RES-002 GPU/NPU 训练任务通过队列调度，默认 FIFO 顺序

- **等级**：MUST
- **触发场景**：包含 GPU 或 NPU 资源申请的训练任务提交时；资源调度器分配 GPU 资源时
- **规则描述**：所有包含 GPU 或 NPU 资源申请的训练任务必须经过任务队列进行调度，不允许绕过队列直接占用 GPU 资源；默认调度顺序为先进先出（FIFO）；支持通过优先级配置调整调度顺序，但优先级的变更须由有权限的管理员操作。
- **校验逻辑**：GPU 训练任务提交后，系统强制将其加入对应资源池的任务队列；调度器按照队列顺序（FIFO 或优先级）分配资源；任务不得直接绑定特定 GPU 节点绕过调度器（仅超级管理员可操作绑定）。
- **异常处理**：尝试绕过队列直接提交时，系统拒绝并提示"GPU 任务必须通过调度队列提交"；队列满时（超过最大长度），拒绝新任务入队并提示"任务队列已满，请稍后重试或联系管理员扩展队列容量"。
- **配置项**：调度策略（FIFO / 优先级队列 / 公平调度，默认 FIFO）；单任务最大 GPU 数量限制；队列最大长度；优先级变更权限角色
- **交互终端**：PC / Interface / 系统自动

### RES-003 存储池删除前须确保所有引用数据集已迁移

- **等级**：MUST
- **触发场景**：存储管理员执行存储池删除操作时
- **规则描述**：存储池删除前，系统必须检查该存储池上是否存在未迁移的数据集、模型文件或训练任务输出；若存在引用，则禁止删除存储池，须先将所有数据迁移到其他存储池后方可执行删除。
- **校验逻辑**：删除操作触发时，系统查询数据集存储位置表、模型文件存储表和任务输出记录，统计仍引用该存储池的对象数量；若数量 > 0，则阻断删除，展示引用清单（对象名称、类型、大小）。
- **异常处理**：校验失败时，系统提示"存储池 [名称] 仍有 [N] 个对象未迁移，请先完成数据迁移后再删除，或使用'迁移并删除'向导"；提供一键迁移向导辅助操作。
- **配置项**：无
- **交互终端**：PC（存储管理员）

### RES-004 基础镜像须通过安全扫描后方可用于训练和推理

- **等级**：MUST
- **触发场景**：镜像管理员上传或导入新镜像后；镜像被选用于训练任务或推理服务时
- **规则描述**：所有用于训练任务、开发环境、推理服务部署的基础镜像，必须完成安全漏洞扫描且结果为通过（无高危漏洞）；未扫描或存在高危漏洞的镜像不得出现在可选镜像列表中，不得被任务引用。
- **校验逻辑**：镜像上传完成后，系统自动触发安全扫描任务；扫描完成前镜像状态为"扫描中"，不可用；扫描完成后按结果更新状态：PASSED（无高危漏洞）或 FAILED（存在高危漏洞）；任务创建时仅展示 PASSED 状态镜像。
- **异常处理**：扫描结果为 FAILED 时，系统标记镜像为"不可用"，通知镜像管理员处理（修复漏洞后重新上传）；扫描服务不可用时，镜像保持"待扫描"状态，系统告警运维人员；不允许跳过扫描强制使用（即便是管理员）。
- **配置项**：安全扫描工具配置（Trivy / Clair / 其他）；拦截漏洞等级（Critical 必须拦截，High 可配置为拦截或警告，默认均拦截）；扫描触发方式（上传后自动 / 手动触发）
- **交互终端**：系统自动（扫描）/ PC（镜像管理员查看结果）

### RES-005 闲置开发环境超时后自动释放资源

- **等级**：CONFIG
- **触发场景**：开发环境（Notebook / VSCode / SSH）长时间无活跃操作时；系统资源回收定时任务执行时
- **规则描述**：开发环境在超过配置的最大空闲时限内无用户活跃操作（无代码执行、无文件保存、无输入），系统自动停止并释放该开发环境占用的计算资源；资源释放前向用户发送预警通知。
- **校验逻辑**：系统定时检测开发环境的最后活跃时间戳；当 `当前时间 - lastActiveTime > idleTimeout` 时，触发警告通知（提前 10 分钟）；用户无响应后执行自动停止，释放 CPU/GPU/内存资源；开发环境代码和数据不删除（仅停止容器）。
- **异常处理**：自动停止失败时（容器状态异常），系统记录失败日志，告警运维人员手动处理；资源释放后用户重启开发环境时，系统提示"环境因空闲超时已被停止，请重新启动"。
- **配置项**：开发环境最大空闲时限（BU 级可配置，默认 2 小时）；预警提前通知时间（默认 10 分钟）；空闲检测间隔（默认 5 分钟）；是否对特定用户组豁免（如管理员角色）
- **交互终端**：系统自动 / PC（用户收到预警后可延期）

### RES-006 资源池配额调整须有审批记录

- **等级**：SHOULD
- **触发场景**：超级管理员或 BU 管理员修改 BU 资源配额（增减 CPU/内存/GPU 配额）时
- **规则描述**：对 BU 资源配额的调整（增加或减少）应经过审批流程，变更记录须包含申请原因、变更前后配额值、审批人和审批时间；防止配额被随意调整影响其他 BU 的资源保障。
- **校验逻辑**：配额变更操作触发审批工单（BU 管理员申请 → 超级管理员审批）；审批通过后变更生效；超级管理员直接修改时，系统仍要求填写变更原因，并记录在审计日志中（无需审批但需留痕）。
- **异常处理**：审批流程不可用时，允许超级管理员直接操作但须记录原因；配额减少导致当前使用量超过新配额时，系统警告"当前已用量超过新配额，已有任务不受影响，但新任务将受配额限制"，不立即终止运行中任务。
- **配置项**：配额变更审批级别（单级超管审批 / 双级，默认单级）；配额变更原因是否必填（默认必填）
- **交互终端**：PC（申请 / 审批）

### RES-007 镜像版本废弃前须通知使用方并设置保留期

- **等级**：SHOULD
- **触发场景**：镜像管理员将镜像版本标记为"废弃"或触发删除操作时
- **规则描述**：镜像版本废弃前，系统应识别当前正在使用该镜像的开发环境和训练任务，向相关用户发送废弃通知；废弃后镜像须保留不少于配置时长的保留期，期间仍可使用但不可被新任务引用；保留期满后方可物理删除。
- **校验逻辑**：标记废弃操作时，系统查询该镜像的当前引用（活跃开发环境、排队中任务）；有引用时，通知相关用户并展示受影响清单；镜像进入废弃状态后，新建任务时从可选列表中移除；保留期计时器到期后可删除。
- **异常处理**：通知发送失败时，记录日志并重试；保留期内有任务强制使用废弃镜像时（如从排队状态恢复），系统允许但记录警告日志并通知镜像管理员。
- **配置项**：废弃镜像保留期（默认 30 天，可配置）；废弃通知渠道（邮件 / 应用内通知）
- **交互终端**：PC（镜像管理员）/ 系统自动（通知）

### RES-008 单任务最大资源申请量不得超过资源池单任务上限

- **等级**：MUST
- **触发场景**：用户配置训练任务或开发环境，填写资源申请量时
- **规则描述**：单个任务申请的 CPU 核数、内存大小、GPU 数量不得超过所属资源池为单任务配置的最大资源上限，防止单任务独占过多资源影响其他用户公平使用。
- **校验逻辑**：任务提交时，系统检查各项资源申请值是否超过目标资源池的单任务上限配置；若任一项超限，则拒绝提交，提示具体超限项目和允许的最大值。
- **异常处理**：校验失败时，系统提示"申请的 [资源类型] 数量 [X] 超过资源池单任务上限 [Y]，请调整资源配置"；建议用户拆分任务或联系管理员调整上限。
- **配置项**：资源池单任务最大 CPU 核数上限；单任务最大内存上限；单任务最大 GPU 数量上限（均按资源池独立配置）
- **交互终端**：PC / Interface

### RES-009 AI 芯片（GPU/NPU）故障时自动转移任务到备用芯片

- **等级**：SHOULD
- **触发场景**：调度系统检测到 GPU/NPU 芯片故障或驱动异常时；节点健康检查失败时
- **规则描述**：当 AI 芯片（GPU/NPU）出现故障时，系统应尝试将正在运行的任务自动迁移到同资源池内的健康备用芯片；若无可用备用芯片，则将任务重新入队等待资源恢复；同时告警通知运维人员处理故障芯片。
- **校验逻辑**：节点代理定时上报芯片健康状态；检测到故障后，系统将故障节点标记为"不可用"，隔离该节点；对其上运行的任务触发迁移逻辑：查询同池可用节点容量，容量充足则迁移，不足则任务重入队。
- **异常处理**：迁移失败时（无可用资源），任务状态置为"等待"，用户收到通知"所在节点发生故障，任务暂停等待资源恢复"；运维人员收到 CRITICAL 级告警，人工处理后系统自动恢复调度；支持断点续训（参见 MDL-012）。
- **配置项**：节点健康检查间隔（默认 30 秒）；故障判定连续失败次数（默认 3 次）；自动迁移开关（默认开启）
- **交互终端**：系统自动 / PC（运维告警处理）

### RES-010 存储使用率超过告警阈值时通知 BU 管理员

- **等级**：CONFIG
- **触发场景**：存储资源监控定时任务检测到 BU 存储使用量达到告警阈值时
- **规则描述**：当 BU 的存储使用率达到配置的告警阈值时，系统自动向该 BU 管理员发送存储告警通知；告警分级（警告级 / 严重级）提醒管理员及时清理数据或申请扩容，防止存储用尽导致数据写入失败。
- **校验逻辑**：存储监控任务定期统计各 BU 存储使用量；使用率超过警告阈值（默认 80%）时，发送警告级通知；超过严重阈值（默认 95%）时，发送严重级告警并升级通知超级管理员；超过 100% 时，新的数据写入请求被拒绝。
- **异常处理**：存储已满（100%）时，系统拒绝新的数据集上传和训练输出写入，提示"BU 存储配额已满，请清理数据或申请扩容"；确保已有数据不丢失，仅阻断新增写入。
- **配置项**：警告级告警阈值（BU 级可配置，默认 80%）；严重级告警阈值（默认 95%）；告警通知渠道（邮件 / IM）；监控检查间隔（默认 1 小时）
- **交互终端**：系统自动 / PC（BU 管理员处理告警）
