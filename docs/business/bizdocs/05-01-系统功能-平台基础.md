# 05-01 系统功能清单 – 平台基础域（认证 / 用户 / 权限 / 多租户）

> **范围**：认证与会话管理、用户管理、角色与权限管理（RBAC+ABAC）、组织架构与多租户隔离
> **终端覆盖**：PC（管理后台）、Interface（API层）
> **对应场景**：SYS-001 / SYS-002 / SYS-003 / SYS-004

---

## 一、认证与会话管理

- **FUNC-PLT-001 – 账号密码登录**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / Password Login
  - **支撑场景**：用户身份验证与会话建立
  - **主要输入**：用户名/工号、密码
  - **主要输出**：JWT Token（Access Token + Refresh Token）、用户会话信息
  - **约束**：连续失败N次（可配置，如5次）锁定账号30分钟；密码强度规则验证；Token有效期可配置（默认8小时）；Refresh Token用于续期（默认30天），Pr = H（Must Have）

- **FUNC-PLT-002 – 企业SSO单点登录**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / Enterprise SSO
  - **支撑场景**：与企业IDP（身份提供商）集成统一认证
  - **主要输入**：SSO协议配置（OAuth2/SAML）、企业IDP地址、客户端凭证
  - **主要输出**：JWT Token、本地账号（首次SSO时自动创建或手动审核）、登录审计日志
  - **约束**：支持OAuth2 / SAML2协议；首次SSO自动创建本地账号可配置（企业级配置开关）；SSO登录记录审计日志；SSO用户无本地密码，不支持密码重置；禁用SSO账号自动同步删除本地账号，Pr = H（Must Have）

- **FUNC-PLT-003 – Token刷新与续期**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / Token Refresh
  - **支撑场景**：维持长会话且提升安全性
  - **主要输入**：Refresh Token
  - **主要输出**：新的Access Token + 新的Refresh Token（可选旋转）
  - **约束**：Refresh Token采用滑动续期机制（每次刷新延长有效期）；超期强制重登；支持Token旋转策略（旧Token自动失效）；Refresh Token过期时间可配置（默认30天），Pr = H（Must Have）

- **FUNC-PLT-004 – 强制下线**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / Force Logout
  - **支撑场景**：应对账号停用、权限变更、安全事件
  - **主要输入**：用户ID、下线原因
  - **主要输出**：Token黑名单、下线确认、审计日志
  - **约束**：管理员手动操作强制下线；账号停用自动触发强制下线；用户角色/权限变更自动触发强制下线；Token加入黑名单立即生效；黑名单缓存时间不超过Token有效期，Pr = H（Must Have）

- **FUNC-PLT-005 – 密码找回与重置**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / Password Recovery
  - **支撑场景**：用户遗忘密码自助恢复
  - **主要输入**：工号/邮箱、验证码（手机/邮件）
  - **主要输出**：密码重置令牌、重置完成确认
  - **约束**：支持手机短信/邮件验证码（可配置）；验证码有效期5分钟；密码强度规则可配置（长度/大小写/数字/特殊字符）；SSO用户不支持密码重置；重置后强制重登，Pr = H（Must Have）

- **FUNC-PLT-006 – MFA多因素认证**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / Multi-Factor Authentication
  - **支撑场景**：增强账号安全性，尤其是高权限用户
  - **主要输入**：第一因素（密码）、第二因素（TOTP令牌/短信验证码）
  - **主要输出**：认证结果、审计日志
  - **约束**：MFA可选启用；支持TOTP（时间同步一次性密码）/ 短信验证码；高权限操作（如删除数据集、授予权限）可强制要求二次验证；MFA绑定恢复码用于备用登录，Pr = H（Must Have）

- **FUNC-PLT-007 – 登录页与登录态保持**
  - **终端**：PC
  - **所属模块**：Authentication / Login UI & Session Persistence
  - **支撑场景**：用户友好的登录体验
  - **主要输入**：账号/密码、SSO选择、语言偏好
  - **主要输出**：登录成功跳转页面、Session维持
  - **约束**：登录页展示平台Logo、账号密码表单、SSO入口、语言切换（中/英）；登录成功跳转上次访问页面（如无则跳转首页）；支持"记住我"（扩展Session有效期，需用户确认）；Session保持使用HttpOnly Cookie + Token双重机制，Pr = H（Must Have）

- **FUNC-PLT-008 – API Key管理**
  - **终端**：PC / Interface
  - **所属模块**：Authentication / API Key Manager
  - **支撑场景**：程序化接入（数据集同步、模型推理调用等）
  - **主要输入**：API Key名称、权限范围、有效期
  - **主要输出**：API Key（仅生成时可见）、Key配置、使用统计
  - **约束**：API Key绑定用户或服务账号；生成、吊销、有效期可配置；权限范围可指定（如只读/写、限定资源ID）；API Key使用需签名验证；支持Key轮换（生成新Key→过渡期→删除旧Key）；审计日志记录API Key生成/吊销/使用，Pr = H（Must Have）

## 二、用户管理

- **FUNC-PLT-010 – 内部用户创建**
  - **终端**：PC / Interface
  - **所属模块**：User Management / Internal User Creator
  - **支撑场景**：企业内部用户账号新增
  - **主要输入**：姓名、工号、邮箱、所属BU、初始角色
  - **主要输出**：用户ID、账号配置、审计日志
  - **约束**：工号唯一；邮箱唯一；可绑定多个BU（跨BU用户）；初始角色为默认最小权限；创建时发送欢迎邮件含临时密码；超级管理员创建，Pr = H（Must Have）

- **FUNC-PLT-011 – 外部服务账号管理**
  - **终端**：PC / Interface
  - **所属模块**：User Management / Service Account Manager
  - **支撑场景**：系统间程序化调用（如ETL、API集成）
  - **主要输入**：服务账号名、服务描述、权限范围
  - **主要输出**：Service Account、API Key、权限配置
  - **约束**：服务账号仅支持API Key认证，无UI登录；权限范围独立配置，遵循最小权限原则；支持启用/禁用；审计日志记录服务账号创建/修改/删除，Pr = H（Must Have）

- **FUNC-PLT-012 – 用户停用与注销**
  - **终端**：PC / Interface
  - **所属模块**：User Management / User Deactivation
  - **支撑场景**：用户离职、权限收回、合规审计
  - **主要输入**：用户ID、停用原因
  - **主要输出**：用户状态变更、强制下线、告警通知
  - **约束**：停用触发强制下线；检查用户进行中任务并告警（如正在标注、模型训练）；支持恢复（可配置恢复期限）；用户注销为不可逆操作（数据脱敏）；停用/注销前发送确认邮件，Pr = H（Must Have）

- **FUNC-PLT-013 – 用户信息自助修改**
  - **终端**：PC
  - **所属模块**：User Management / Self-Service Profile
  - **支撑场景**：用户自主维护个人信息
  - **主要输入**：联系方式、语言偏好、密码
  - **主要输出**：用户信息更新、验证邮件
  - **约束**：用户可修改：邮箱（需验证）、电话、语言偏好、密码（需旧密码验证）；工号/组织/角色需管理员操作；修改邮箱/密码需二次验证，Pr = H（Must Have）

- **FUNC-PLT-014 – 用户列表查询与导出**
  - **终端**：PC / Interface
  - **所属模块**：User Management / User Query & Export
  - **支撑场景**：用户信息统一查询、合规导出
  - **主要输入**：BU、角色、状态、时间范围、搜索关键词
  - **主要输出**：用户列表、Excel导出文件
  - **约束**：支持按BU/角色/状态/创建时间过滤；支持工号/姓名模糊搜索；导出格式为Excel（含用户基本信息、角色、所属BU、状态）；导出操作记录审计；超级管理员可查看全平台，BU管理员可查看本BU，Pr = H（Must Have）

- **FUNC-PLT-015 – 用户权限复审提醒**
  - **终端**：PC
  - **所属模块**：User Management / Permission Review Reminder
  - **支撑场景**：定期安全治理，收回不必要权限
  - **主要输入**：复审周期配置（默认季度）、高权限阈值
  - **主要输出**：复审提醒邮件、复审工单
  - **约束**：定期提醒管理员复审高权限账号（如超级管理员、数据集所有者）；可配置复审周期（月/季度/年）；复审工单包含权限清单、使用情况分析；支持合并复审（多个用户）；复审结果记录审计，Pr = M（Should Have）

- **FUNC-PLT-016 – BU子管理员用户管理授权**
  - **终端**：PC / Interface
  - **所属模块**：User Management / BU Admin Delegation
  - **支撑场景**：下沉用户管理权限至BU级
  - **主要输入**：BU、管理员用户、授权范围
  - **主要输出**：权限授权配置、审计日志
  - **约束**：超级管理员授权BU子管理员可管理本BU用户（创建、停用、角色分配）；BU子管理员不可跨BU管理；不可超越父角色权限上限；授权/撤销操作记录审计，Pr = H（Must Have）

## 三、角色与权限管理（RBAC + ABAC）

- **FUNC-PLT-020 – 预设角色定义**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / Role Definition
  - **支撑场景**：平台内置角色体系，开箱即用
  - **主要输入**：无（预设角色）
  - **主要输出**：角色清单、权限配置、应用指南
  - **约束**：平台预设6个内置角色，不可删除，可禁用；权限配置可修改，建议保持原意：
  
  | 角色 | 权限范围 | 主要责任 |
  |------|---------|---------|
  | **超级管理员** | 全平台所有功能、跨BU操作、系统配置 | 平台运营、用户管理、系统审计、资源配置 |
  | **BU子管理员** | 本BU内所有功能、BU级配置、用户管理 | BU运营、下属用户管理、BU资源分配 |
  | **数据标注工程师** | 标注任务分配的数据、标注工具、自己的标注结果 | 数据标注、质量自检、标注问题反馈 |
  | **审核工程师** | 标注任务审核、标注结果查看、审核工具 | 标注质量审核、驳回与反馈、审核统计 |
  | **模型训练工程师** | 数据集（有权限的）、模型训练、超参数调优、训练资源 | 模型开发、调优、评估、版本管理 |
  | **模型应用工程师** | 模型部署、模型推理、服务管理、应用监控 | 模型上线、推理服务部署、性能优化 |
  
  Pr = H（Must Have）

- **FUNC-PLT-021 – RBAC权限粒度**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / RBAC Granularity
  - **支撑场景**：分层权限控制，清晰的权限边界
  - **主要输入**：功能权限定义、菜单树、数据权限范围
  - **主要输出**：权限矩阵、权限配置
  - **约束**：三级权限粒度：
    - 第一级：菜单权限（可见菜单项，如"数据管理"、"模型训练"）
    - 第二级：功能权限（具体操作，如"创建数据集"、"启动训练任务"）
    - 第三级：数据权限（资源隔离，如按BU/项目隔离数据集和模型）
  Pr = H（Must Have）

- **FUNC-PLT-022 – ABAC属性权限**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / ABAC Attributes
  - **支撑场景**：细粒度权限，适应复杂业务场景
  - **主要输入**：属性定义（主体属性、资源属性、环境属性）、决策策略
  - **主要输出**：属性配置、ABAC策略
  - **约束**：支持基于属性的细粒度控制：
    - 主体属性：用户角色、所属BU、岗位、安全等级
    - 资源属性：数据集密级（内部/受限/公开）、模型scope（个人/BU/平台）、资源池归属
    - 环境属性：IP范围、访问时间、地点
    - 决策策略：最小权限原则、时间隔离、审批流转
  Pr = H（Must Have）

- **FUNC-PLT-023 – 用户-角色分配**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / User Role Assignment
  - **支撑场景**：动态分配和管理用户角色
  - **主要输入**：用户ID、角色、有效期（可选）
  - **主要输出**：角色分配记录、审计日志
  - **约束**：支持多角色（同时拥有多个角色，如"BU子管理员" + "模型训练工程师"）；角色有效期可选（默认长期）；角色变更触发强制下线+审计日志；超级管理员可分配所有角色，BU管理员可分配BU内角色；权限取用户所有角色权限集合（并集），Pr = H（Must Have）

- **FUNC-PLT-024 – 自定义角色**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / Custom Role Manager
  - **支撑场景**：适应特殊业务场景
  - **主要输入**：角色名、权限集合、应用范围
  - **主要输出**：自定义角色配置、权限检查结果
  - **约束**：在预设角色基础上增减权限；不可超越父角色权限上限；自定义角色绑定创建人及部门（影响可见性）；变更自定义角色需受影响用户重新授权；删除自定义角色前检查关联用户，Pr = M（Should Have）

- **FUNC-PLT-025 – 权限实时校验**
  - **终端**：Interface / API
  - **所属模块**：Permission Management / Permission Validation
  - **支撑场景**：每个API请求都需验证用户权限
  - **主要输入**：用户身份、请求资源、操作类型
  - **主要输出**：权限校验结果（允许/拒绝）
  - **约束**：API层Token验证+权限检查+数据行租户过滤；权限检查缓存有效期≤5分钟；权限缓存更新采用被动失效+定时刷新；权限拒绝返回403，记录审计日志，Pr = H（Must Have）

- **FUNC-PLT-026 – 数据资产权限申请流程**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / Data Permission Request
  - **支撑场景**：用户申请访问受限数据（数据集/模型）
  - **主要输入**：资源ID、申请理由、期限
  - **主要输出**：申请单、审批工单、权限授权
  - **约束**：申请流程：用户申请→资源所有者审批→权限授权（或拒绝）；可指定期限（如30天临时访问）；申请理由可配置为必填；审批可设置自动过期时间；记录权限使用期限及历史，Pr = H（Must Have）

- **FUNC-PLT-027 – 最小权限原则执行**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / Least Privilege Enforcement
  - **支撑场景**：安全治理，降低权限滥用风险
  - **主要输入**：用户权限配置、系统策略
  - **主要输出**：权限评估结果、提醒通知
  - **约束**：新用户创建后权限为零（无菜单访问）；所有权限申请制，禁止默认高权限；定期扫描异常权限组合，发送告警；权限审批需记录批准人和批准时间；高权限操作（如删除数据）需二次审批或MFA验证，Pr = H（Must Have）

- **FUNC-PLT-028 – 权限变更申请与审批**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / Permission Change Request
  - **支撑场景**：权限变更需审批，确保合规
  - **主要输入**：变更内容（新增/删除/修改权限）、申请理由
  - **主要输出**：申请工单、审批流程、变更记录
  - **约束**：权限变更流程：发起申请→BU管理员审批→超级管理员确认（可选）；可配置为单级或两级审批；权限变更记录完整的版本历史；支持工单流程集成（如与审批系统对接）；变更后触发强制下线重新认证，Pr = M（Should Have）

- **FUNC-PLT-029 – 公共资源访问控制**
  - **终端**：PC / Interface
  - **所属模块**：Permission Management / Public Resource Access
  - **支撑场景**：平台内共享资源对全用户开放，细粒度写权限控制
  - **主要输入**：资源ID、资源属性、用户权限
  - **主要输出**：访问决策、权限评估
  - **约束**：公共资源（如平台模型、推荐数据集）对所有认证用户可读；写权限受ABAC控制（如仅所有者可写）；修改公共资源需审批或工单流程；公共资源不能被删除，仅所有者可下架；访问公共资源不需额外申请，但修改需权限，Pr = H（Must Have）

## 四、组织架构与多租户隔离

- **FUNC-PLT-031 – 组织树管理**
  - **终端**：PC / Interface
  - **所属模块**：Organization Management / Org Tree Manager
  - **支撑场景**：企业多级组织结构纳管
  - **主要输入**：组织节点（集团/BU/项目）、层级关系、属性配置
  - **主要输出**：组织树、节点配置、关联用户
  - **约束**：三级树结构：集团（顶层）→ BU（中层，多个）→ 项目（叶层，多个）；超级管理员管理组织树；支持增删改查；删除BU前检查关联用户和数据；BU编码唯一；项目编码在BU内唯一，Pr = H（Must Have）

- **FUNC-PLT-032 – 租户上下文注入**
  - **终端**：Interface / API
  - **所属模块**：Multi-Tenancy / Tenant Context Injection
  - **支撑场景**：多租户数据隔离的技术基础
  - **主要输入**：用户身份、JWT Token
  - **主要输出**：Tenant Context（tenantId/buId/projectId）
  - **约束**：JWT Token携带tenantId/buId；所有查询自动附加租户过滤条件（WHERE tenant_id = ?）；超级管理员可跨租户操作（需显式指定）；上下文注入在认证后立即执行，影响整个请求周期；支持审计日志记录跨租户操作，Pr = H（Must Have）

- **FUNC-PLT-033 – BU（租户）配置**
  - **终端**：PC / Interface
  - **所属模块**：Multi-Tenancy / BU Configuration
  - **支撑场景**：BU级个性化配置
  - **主要输入**：BU编码、名称、所属集团、时区、默认语言、配置参数
  - **主要输出**：BU配置、继承配置
  - **约束**：BU编码/名称唯一；BU级配置可覆盖全局默认值（如时区、语言、资源配额告警阈值、标注配置）；配置变更不影响现有数据；超级管理员或BU管理员可修改；支持配置版本管理，Pr = H（Must Have）

- **FUNC-PLT-034 – 项目隔离**
  - **终端**：PC / Interface
  - **所属模块**：Multi-Tenancy / Project Isolation
  - **支撑场景**：BU内细粒度资源隔离
  - **主要输入**：项目编码、名称、所属BU、项目成员
  - **主要输出**：项目配置、项目成员管理
  - **约束**：数据集、模型、训练任务按项目隔离；仅项目成员可访问项目资源；项目成员由BU管理员分配；删除项目前检查关联资源；项目成员离职时需移除，Pr = H（Must Have）

- **FUNC-PLT-035 – 跨BU用户绑定**
  - **终端**：PC / Interface
  - **所属模块**：Multi-Tenancy / Cross-BU User Binding
  - **支撑场景**：支持用户在多个BU工作
  - **主要输入**：用户ID、BU列表、BU内角色
  - **主要输出**：用户-BU映射、身份切换
  - **约束**：一个用户可绑定多个BU；切换BU时重签新Token（携带不同buId）；各BU角色独立配置（如在YFI是"BU子管理员"，在YFS是"数据标注工程师"）；用户可在不同BU间切换身份；切换记录审计日志，Pr = H（Must Have）

- **FUNC-PLT-036 – 平台模型与团队模型隔离**
  - **终端**：PC / Interface
  - **所属模块**：Model Management / Model Scope Isolation
  - **支撑场景**：区分通用模型与团队私有模型
  - **主要输入**：模型scope属性（平台/BU/项目/私人）、访问控制列表
  - **主要输出**：模型可见性、访问权限
  - **约束**：平台模型（官方训练）全局可读，不可修改（写权限受限）；团队模型（BU/项目自训练）按BU/项目ACL控制，仅成员可见；私人模型仅所有者可见；模型scope变更需管理员审批；模型版本化管理，旧版本不影响权限，Pr = H（Must Have）

- **FUNC-PLT-037 – 跨BU数据集授权**
  - **终端**：PC / Interface
  - **所属模块**：Dataset Management / Cross-BU Dataset Authorization
  - **支撑场景**：支持数据共享，同时保持BU数据控制权
  - **主要输入**：数据集ID、目标BU、授权权限（读/写）、期限
  - **主要输出**：授权记录、权限生效
  - **约束**：BU子管理员可授权本BU数据集给其他BU访问；记录授权关系（谁授权、何时、权限范围）；授权有效期可配置；接收方BU管理员需确认方可生效；授权撤销立即生效；数据所有权不变（仍属授权方），Pr = H（Must Have）

- **FUNC-PLT-038 – BU级系统配置隔离**
  - **终端**：PC / Interface
  - **所属模块**：Multi-Tenancy / BU Configuration Isolation
  - **支撑场景**：支持BU个性化系统配置
  - **主要输入**：配置项、配置值、应用范围
  - **主要输出**：BU级配置、继承链
  - **约束**：支持BU级覆盖的全局配置项：
    - 通知偏好（邮件/短信频率、收件人）
    - 资源配额告警阈值（存储、计算、标注数量）
    - 标注配置（最大样本数、审核规则）
    - 数据下载开关（允许/禁止下载指定数据集）
    - 登录密码策略（强度、过期时间）
    - 备份策略（频率、保留期限）
  BU级配置优先级高于全局配置；配置变更不影响已有资源，但对新资源生效；超级管理员管理全局配置，BU管理员管理本BU配置，Pr = H（Must Have）
