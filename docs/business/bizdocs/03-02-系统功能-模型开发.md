# 03-02 系统功能清单 – 模型开发域

> **范围**：模型调优、模型评估、模型管理、模型工程化、模型开发环境、训练任务等全流程
> **组织方式**：按子模块分组，每功能点含终端、模块路径、支撑场景、输入输出、约束

---

## 一、模型调优

- **FUNC-MODEL-001 – 预训练模型选择**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Pretrained Model Selector
  - **支撑场景**：快速开始模型开发
  - **主要输入**：场景类型、任务类型
  - **主要输出**：可用模型列表、模型详情
  - **约束**：按场景类型/任务类型筛选，Pr = H（Must Have）

- **FUNC-MODEL-002 – 数据集拆分**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Dataset Splitter
  - **支撑场景**：训练数据划分
  - **主要输入**：原始数据集、拆分比例
  - **主要输出**：训练集、验证集、测试集
  - **约束**：支持训练/验证集比例配置，Pr = H（Must Have）

- **FUNC-MODEL-003 – 超参数手动配置**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Hyperparameter Config
  - **支撑场景**：精细化模型调优
  - **主要输入**：学习率、批大小、Epoch、优化器等参数
  - **主要输出**：超参数配置文件
  - **约束**：支持学习率、批大小、Epoch等自定义配置，Pr = H（Must Have）

- **FUNC-MODEL-004 – 自动超参数调优**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / AutoML Engine
  - **支撑场景**：降低调优门槛
  - **主要输入**：搜索空间定义、优化目标
  - **主要输出**：最优超参数、调优历史
  - **约束**：AutoML超参数自动搜索优化，Pr = M（Should Have）

- **FUNC-MODEL-005 – 调优任务资源申请**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Resource Allocator
  - **支撑场景**：计算资源配置
  - **主要输入**：资源类型、数量
  - **主要输出**：资源分配结果、任务配置
  - **约束**：支持CPU/GPU/NPU申请，Pr = H（Must Have）

- **FUNC-MODEL-006 – 训练过程监控**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Training Monitor
  - **支撑场景**：训练进度实时跟踪
  - **主要输入**：训练任务ID
  - **主要输出**：实时日志、资源使用、训练曲线
  - **约束**：实时日志、资源使用、Tensorboard可视化，Pr = H（Must Have）

- **FUNC-MODEL-007 – 训练日志持久化**
  - **终端**：PC / Interface / Batch
  - **所属模块**：Model Training / Log Storage
  - **支撑场景**：训练历史保存查询
  - **主要输入**：训练日志数据
  - **主要输出**：日志档案、查询索引
  - **约束**：日志存储与查询，Pr = H（Must Have）

- **FUNC-MODEL-008 – 调优任务管理**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Task Manager
  - **支撑场景**：任务生命周期管理
  - **主要输入**：任务过滤条件、操作命令
  - **主要输出**：任务列表、操作结果
  - **约束**：支持搜索、启动、终止、删除，Pr = H（Must Have）

- **FUNC-MODEL-009 – 调优产物导出**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Model Exporter
  - **支撑场景**：训练模型保存
  - **主要输入**：训练任务ID、导出格式
  - **主要输出**：模型文件、配置元数据
  - **约束**：导出训练完成的模型文件，Pr = H（Must Have）

- **FUNC-MODEL-010 – 调优产物推送边端**
  - **终端**：PC / Interface
  - **所属模块**：Model Training / Edge Deployment
  - **支撑场景**：模型直接部署到边端
  - **主要输入**：模型版本、目标边端服务器
  - **主要输出**：部署配置、推送状态
  - **约束**：直接推送到指定边端服务器，Pr = H（Must Have）

## 八、开发环境多操作入口

- **FUNC-MODEL-065 – 7 种开发操作入口**
  - **终端**：PC / Interface
  - **所属模块**：Development Environment / Operation Entry
  - **支撑场景**：开发者灵活接入开发环境
  - **主要输入**：开发任务 ID、操作类型选择
  - **主要输出**：对应入口会话（IDE/Terminal/SSH/镜像任务）
  - **约束**：支持 Web IDE / VSCode 远程连接 / SSH / 打开终端 / 制作镜像 / 分享环境 / 停止 共 7 种操作；状态机控制各入口可用性，Pr = H（Must Have）

- **FUNC-MODEL-066 – 零门槛调优 3 步向导**
  - **终端**：PC / Interface
  - **所属模块**：Development Environment / Fine-tuning Wizard
  - **支撑场景**：无代码快速启动微调任务
  - **主要输入**：Step1 数据集（含 80/10/10 分割比例三联输入）；Step2 预训练模型；Step3 超参数（lr=0.001, batch_size=16, epochs=10, optimizer=AdamW）
  - **主要输出**：FineTuningJob 任务 ID、任务配置摘要
  - **约束**：三步向导提交后生成 FineTuningJob；数据集分割比例可视化三联输入，支持手动调整，Pr = H（Must Have）

  - **主要输入**：失败任务 ID、日志内容
  - **主要输出**：根因分类（资源不足/数据问题/代码错误/框架兼容）、修复建议、追问对话
  - **约束**：AI 诊断助手自动分析日志并给出根因和建议；支持用户追问，Pr = M（Should Have）
