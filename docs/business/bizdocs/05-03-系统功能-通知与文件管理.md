# 05-03 系统功能清单 – 通知与文件管理

> **范围**：通知中心、消息推送、文件上传下载、文件存储分层、权限控制、回收站管理、存储配额等全流程
> **对应场景**：SYS-007 / SYS-008 / SYS-011

---

## 一、通知与消息中心

- **FUNC-NOTIFY-001 – 站内消息中心**
  - **终端**：PC / Interface
  - **所属模块**：Notification / Message Center
  - **支撑场景**：用户通知消息统一管理
  - **主要输入**：消息过滤条件、操作命令
  - **主要输出**：消息列表、消息详情、操作结果
  - **约束**：支持消息列表展示、已读/未读标记、批量已读、删除操作；顶部导航栏消息图标展示未读数角标；按消息类型分类过滤（任务通知、申请审批、系统公告等），Pr = H（Must Have）

- **FUNC-NOTIFY-002 – 通知触发场景清单**
  - **终端**：PC / Interface / System
  - **所属模块**：Notification / Trigger Engine
  - **支撑场景**：自动化通知派发

| 场景 | 触发条件 | 接收人 | 优先级 | 渠道 |
|------|--------|--------|--------|------|
| 训练任务完成/失败 | 训练任务执行完成或失败 | 提交任务的训练工程师 | 高 | 站内/邮件/IM |
| 模型调优任务完成/失败 | 调优任务执行完成或失败 | 提交任务的模型工程师 | 高 | 站内/邮件/IM |
| 数据集申请待审批 | 用户申请使用受限数据集 | BU管理员/超级管理员 | 高 | 站内/邮件/IM |
| 数据集申请审批结果 | 申请被批准或驳回 | 申请人 | 高 | 站内/邮件/IM |
| 标注任务待审核 | 标注工作提交完成 | 审核工程师 | 中 | 站内/邮件 |
| 标注任务审核结果 | 审核通过或驳回 | 标注工程师 | 中 | 站内/邮件 |
| 模型下发边端成功/失败 | 模型推送到边端服务器完成 | 模型应用工程师 | 高 | 站内/邮件/IM |
| 推理服务异常/健康检查失败 | 推理服务故障或无响应 | 负责人 | 高 | 站内/邮件/IM |
| 资源使用超阈值告警 | CPU/GPU/存储使用率超过配置阈值 | BU管理员 | 中 | 站内/邮件/IM |
| 存储容量接近上限 | 存储使用量超过配额80% | BU管理员 | 中 | 站内/邮件/IM |
| 数据集权限到期提醒 | 数据集使用权限即将过期（提前7天）| 使用者 | 中 | 站内/邮件 |
| 系统维护通知 | 定时/临时系统维护 | 全体用户 | 低 | 站内 |
| 账号权限变更 | 用户权限被新增/修改/撤销 | 当事人 | 高 | 站内/邮件/IM |

  - **主要输入**：业务事件触发、通知参数配置
  - **主要输出**：通知消息、派发记录
  - **约束**：覆盖13类通知场景，支持多优先级和多渠道配置，Pr = H（Must Have）

- **FUNC-NOTIFY-003 – 邮件通知**
  - **终端**：PC / Interface / Backend
  - **所属模块**：Notification / Email Notifier
  - **支撑场景**：邮件方式通知用户
  - **主要输入**：邮件服务器配置、收件地址、邮件内容
  - **主要输出**：邮件发送状态、发送日志
  - **约束**：支持邮件服务器配置（SMTP参数）；用户可自助开启/关闭邮件通知；支持按通知类型分别控制邮件开关；高优先级通知强制发送邮件（不受用户设置影响）；邮件模板支持国际化（中英文），Pr = H（Must Have）

- **FUNC-NOTIFY-004 – 企业IM集成**
  - **终端**：PC / Interface / Backend
  - **所属模块**：Notification / IM Integration
  - **支撑场景**：消息同步到企业即时通讯工具
  - **主要输入**：IM平台Webhook配置、消息内容
  - **主要输出**：IM消息派发状态、消息链接
  - **约束**：支持配置对接企业微信/钉钉/飞书Webhook；管理员全局配置IM集成参数；用户可选择是否同步通知到IM；IM消息包含平台跳转链接（便于用户快速跳转到相关页面），Pr = M（Should Have）

- **FUNC-NOTIFY-005 – 通知偏好设置**
  - **终端**：PC / Interface
  - **所属模块**：Notification / Preference Manager
  - **支撑场景**：个性化通知配置
  - **主要输入**：通知类型、接收渠道选择
  - **主要输出**：用户偏好配置
  - **约束**：用户自助配置各类通知的接收状态（开启/关闭）；用户可分别选择接收渠道（站内/邮件/IM）；BU管理员可设置BU级默认偏好作为新用户的初始配置，Pr = H（Must Have）

- **FUNC-NOTIFY-006 – 任务进度实时推送**
  - **终端**：PC / Interface
  - **所属模块**：Notification / Progress Stream
  - **支撑场景**：长时任务进度实时反馈
  - **主要输入**：任务ID、进度数据
  - **主要输出**：实时进度通知、进度条更新
  - **约束**：支持WebSocket/SSE长连接实时推送；页面内显示任务进度条（完成百分比、预计剩余时间）；浏览器离开页面后降级为任务完成时的通知；支持训练任务、标注任务、推理批处理等长时任务，Pr = H（Must Have）

- **FUNC-NOTIFY-007 – 系统公告**
  - **终端**：PC / Interface
  - **所属模块**：Notification / Announcement Manager
  - **支撑场景**：平台级信息发布
  - **主要输入**：公告内容、发布配置、生效期限
  - **主要输出**：公告展示、发布记录
  - **约束**：超级管理员发布全平台公告；支持指定BU范围或全体用户；公告在首页/消息中心置顶显示；支持定时发布（指定发布时间）；支持自动失效（指定失效时间或有效期），Pr = M（Should Have）

- **FUNC-NOTIFY-008 – 资源告警通知**
  - **终端**：PC / Interface / Backend
  - **所属模块**：Notification / Alert Engine
  - **支撑场景**：异常情况自动告警
  - **主要输入**：监控指标、告警规则、阈值配置
  - **主要输出**：告警消息、告警日志
  - **约束**：资源使用率超阈值自动发送告警；任务排队超时自动告警；边端服务器掉线自动告警；告警等级分为Info/Warning/Critical三级；管理员可配置告警阈值和接收人，Pr = M（Should Have）

- **FUNC-NOTIFY-009 – 通知历史记录**
  - **终端**：PC / Interface
  - **所属模块**：Notification / History Archive
  - **支撑场景**：通知消息归档查询
  - **主要输入**：时间范围、消息类型过滤
  - **主要输出**：历史记录列表、消息详情
  - **约束**：保留所有通知的历史记录，支持按时间范围查询；系统通知保留90天；任务相关通知随任务记录保留（任务删除后仍保留一定时间），Pr = M（Should Have）

- **FUNC-NOTIFY-010 – 通知已读/未读统计**
  - **终端**：PC / Interface
  - **所属模块**：Notification / Statistics Dashboard
  - **支撑场景**：通知处理情况统计
  - **主要输入**：时间范围、统计维度
  - **主要输出**：统计数据、趋势图表
  - **约束**：首页展示当前用户未读通知数；BU管理员可查看BU内通知处理情况统计（消息数、已读率、响应时间等），Pr = M（Should Have）

## 二、文件管理

- **FUNC-FILE-001 – 文件上传**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Upload Handler
  - **支撑场景**：数据集、模型、镜像等文件上传
  - **主要输入**：本地文件、上传参数、资产元数据
  - **主要输出**：文件ID、上传进度、存储路径
  - **约束**：支持分块上传机制（处理大文件）；断点续传（网络中断后可恢复）；上传进度实时显示（百分比、速度、剩余时间）；按资产类型校验文件格式（数据集/模型/镜像等有不同的格式限制）；可选病毒扫描功能；单文件大小上限可由平台管理员配置，默认10GB，Pr = H（Must Have）

- **FUNC-FILE-002 – 文件存储分层**
  - **终端**：Backend / System
  - **所属模块**：File Management / Storage Tier Manager
  - **支撑场景**：成本优化与性能平衡
  - **主要输入**：文件访问模式、存储策略
  - **主要输出**：存储层分配、迁移日志
  - **约束**：热数据存储在高性能存储（NFS/对象存储）；冷数据自动迁移到低成本存储（归档存储）；文件路径由平台统一管理和虚拟化，用户不直接访问底层存储路径；支持自定义迁移策略（访问时间、文件大小、资产类型等条件），Pr = M（Should Have）

- **FUNC-FILE-003 – 文件下载与权限控制**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Download Manager
  - **支撑场景**：安全的文件获取
  - **主要输入**：文件ID、用户身份、下载请求
  - **主要输出**：下载链接、文件内容、下载记录
  - **约束**：下载前进行权限验证（确保用户有访问权限）；大文件异步打包下载（>100MB时）；生成临时签名URL（有效期可配置，默认1小时）；记录所有下载操作审计日志（用户、时间、文件、IP等），Pr = H（Must Have）

- **FUNC-FILE-004 – 数据集文件管理**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Dataset File Manager
  - **支撑场景**：数据集内部文件组织
  - **主要输入**：数据集ID、文件列表
  - **主要输出**：文件目录树、文件详情
  - **约束**：支持数据集文件的组织目录结构（支持多级目录）；支持文件列表预览（表格展示文件名、大小、修改时间等）；按文件类型过滤展示；支持样例文件预览（图片/音频/视频显示缩略图，文本显示前N行），Pr = H（Must Have）

- **FUNC-FILE-005 – 模型文件管理**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Model File Manager
  - **支撑场景**：模型文件版本管理
  - **主要输入**：模型版本、文件内容
  - **主要输出**：文件清单、版本对比、文件详情
  - **约束**：模型权重文件/配置文件/评估报告统一存储管理；支持不同模型版本间文件差异对比（显示新增/删除/修改的文件）；支持模型文件格式说明（.pt/.onnx/.bin/.pb等格式的描述和用途），Pr = M（Should Have）

- **FUNC-FILE-006 – 镜像文件管理**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Image Registry
  - **支撑场景**：容器镜像分发管理
  - **主要输入**：镜像Dockerfile、远程仓库地址、镜像标签
  - **主要输出**：镜像ID、仓库配置、镜像列表
  - **约束**：容器镜像文件的统一存储与分发；镜像仓库管理（支持多个仓库）；支持从本地上传Dockerfile构建镜像，也支持从远程仓库（Docker Hub、Harbor等）拉取镜像；可选镜像扫描功能（检测安全漏洞、恶意软件等），Pr = M（Should Have）

- **FUNC-FILE-007 – 标注结果文件管理**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Annotation Result Manager
  - **支撑场景**：标注产物导出与版本管理
  - **主要输入**：标注任务ID、导出格式、数据集范围
  - **主要输出**：标注结果文件、导出配置
  - **约束**：支持多种标注导出格式（COCO/YOLO/VOC/自定义JSON等）；支持按标注任务或数据集维度导出；标注文件版本与数据集版本关联（记录标注对应的数据集版本），Pr = M（Should Have）

- **FUNC-FILE-008 – 回收站与文件恢复**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Recycle Bin
  - **支撑场景**：误删恢复保护
  - **主要输入**：删除操作、恢复请求
  - **主要输出**：回收站文件列表、恢复结果
  - **约束**：删除的文件先进入回收站（软删除），不物理删除；回收站文件保留N天后自动物理删除（N天数可配置，默认30天）；管理员可手动从回收站恢复文件；物理删除操作必须记录详细审计日志（删除人、时间、文件、原因等），Pr = M（Should Have）

- **FUNC-FILE-009 – 存储配额管理**
  - **终端**：PC / Interface
  - **所属模块**：File Management / Storage Quota Manager
  - **支撑场景**：存储资源控制
  - **主要输入**：BU/项目、配额大小
  - **主要输出**：配额配置、使用统计
  - **约束**：按BU/项目分配存储配额；实时显示已用/剩余空间（进度条或百分比）；超配额时阻止新文件上传并触发告警通知；管理员可灵活调整配额大小；支持配额预警（如使用量超过80%时提前告警），Pr = H（Must Have）

---

> 约束：所有清单仅基于 SOR V1.0 内容抽取；优先级 H=Must Have，M=Should Have，L=Could Live Without。
