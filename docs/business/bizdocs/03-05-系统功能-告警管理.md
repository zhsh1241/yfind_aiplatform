# 03-05 系统功能清单 – 告警管理

> **补入日期**：2026-05-09

## 一、告警触发与采集

- **FUNC-ALT-001 – 告警规则配置**
  - **终端**：PC / Interface
  - **所属模块**：Alert Management / Rule Engine
  - **支撑场景**：管理员定义告警触发条件
  - **主要输入**：指标类型（GPU利用率/任务失败率/服务响应时延/队列积压数）、阈值、持续时间、告警级别
  - **主要输出**：告警规则列表
  - **约束**：支持严重/警告/信息三个级别；Pr = H（Must Have）

- **FUNC-ALT-002 – 告警自动触发**
  - **终端**：后台服务 / Interface（只读展示）
  - **所属模块**：Alert Management / Trigger Engine
  - **支撑场景**：系统指标超阈值时自动生成告警事件
  - **主要输入**：监控指标数据流
  - **主要输出**：告警事件（含触发时间、来源、级别、描述）
  - **约束**：告警生成后实时推送到告警中心；Pr = H（Must Have）

---

## 二、告警中心页面

- **FUNC-ALT-010 – 告警统计概览**
  - **终端**：PC / Interface
  - **所属模块**：Alert Center / Dashboard
  - **支撑场景**：运维人员快速了解当前告警态势
  - **主要输入**：告警事件数据库
  - **主要输出**：严重/警告/信息三类数量统计卡
  - **约束**：统计卡数字实时刷新；Pr = H（Must Have）

- **FUNC-ALT-011 – 告警列表与筛选**
  - **终端**：PC / Interface
  - **所属模块**：Alert Center / List View
  - **支撑场景**：快速定位特定告警
  - **主要输入**：筛选条件（级别/来源/状态/时间范围/关键词）
  - **主要输出**：分页告警列表
  - **约束**：支持多维筛选组合；Pr = H（Must Have）

- **FUNC-ALT-012 – 告警详情查看**
  - **终端**：PC / Interface
  - **所属模块**：Alert Center / Detail View
  - **支撑场景**：运维人员深入了解告警上下文
  - **主要输入**：告警事件ID
  - **主要输出**：详情弹窗（元信息4宫格：触发时间/来源/级别/影响范围、建议处理方案）
  - **约束**：点击告警行或「详情」按钮触发；Pr = H（Must Have）

---

## 三、告警处理流程

- **FUNC-ALT-020 – 告警确认处理**
  - **终端**：PC / Interface
  - **所属模块**：Alert Center / Action
  - **支撑场景**：运维人员处理并关闭告警
  - **主要输入**：处理备注（可选文本）、确认操作
  - **主要输出**：告警状态变更（处理中→已处理）、处理记录
  - **约束**：确认处理后告警行内状态实时更新，无需页面刷新；Pr = H（Must Have）

- **FUNC-ALT-021 – 建议处理方案展示**
  - **终端**：PC / Interface
  - **所属模块**：Alert Center / AI Suggestion
  - **支撑场景**：辅助运维人员快速定位根因
  - **主要输入**：告警类型、历史处理记录
  - **主要输出**：蓝色高亮建议方案文本区（含根因分析、处理步骤）
  - **约束**：方案由规则引擎或AI生成；Pr = M（Should Have）

- **FUNC-ALT-022 – 告警升级与通知**
  - **终端**：后台服务 / Interface（配置）
  - **所属模块**：Alert Center / Escalation
  - **支撑场景**：严重告警超时未处理时自动升级通知
  - **主要输入**：升级规则（超时时长、通知渠道、接收人）
  - **主要输出**：升级通知（邮件/企业IM）
  - **约束**：与 05-03 通知系统集成；Pr = M（Should Have）
