# 05-04 系统功能清单 – 集成治理

> **范围**：API开放、身份认证集成、第三方推理服务纳管、边端集成等企业级集成能力
> **组织方式**：按子模块分组，每功能点含终端、模块路径、支撑场景、输入输出、约束
> **对应场景**：SYS-012

---

## 一、API网关与开放接口

- **FUNC-INTG-001 – REST API开放**
  - **终端**：API / 后端
  - **所属模块**：Integration / API Gateway
  - **支撑场景**：平台核心功能对外开放
  - **主要输入**：API请求（数据集查询、模型推理调用、任务状态查询等）
  - **主要输出**：API响应、OpenAPI文档
  - **约束**：支持JSON/Protobuf格式；版本化API（v1/v2）；自动生成OpenAPI/Swagger文档；Pr = H（Must Have）

- **FUNC-INTG-002 – API Key认证与管理**
  - **终端**：API / 后端
  - **所属模块**：Integration / API Auth Manager
  - **支撑场景**：调用者身份认证与权限管理
  - **主要输入**：用户/服务账号、权限范围配置
  - **主要输出**：API Key、权限策略、访问记录
  - **约束**：支持Key生成/吊销/有效期管理；与用户/服务账号绑定；访问记录审计；Pr = H（Must Have）

- **FUNC-INTG-003 – API限流与配额**
  - **终端**：API / 后端
  - **所属模块**：Integration / Rate Limiter
  - **支撑场景**：平台资源保护与公平使用
  - **主要输入**：API Key、限流规则配置
  - **主要输出**：限流决策、配额状态
  - **约束**：按API Key/IP/BU配置QPS限流；支持日/月调用量限制；超限返回429；触发告警；Pr = H（Must Have）

- **FUNC-INTG-004 – API调用统计与监控**
  - **终端**：PC / Interface
  - **所属模块**：Integration / API Metrics
  - **支撑场景**：API运营数据统计与问题诊断
  - **主要输入**：时间范围、统计维度
  - **主要输出**：调用量、错误率、延迟统计、Dashboard可视化
  - **约束**：实时调用量/错误率/延迟统计；按API/调用方/时间维度聚合；可视化Dashboard；异常自动告警；Pr = H（Must Have）

- **FUNC-INTG-005 – Webhook事件推送**
  - **终端**：API / 后端
  - **所属模块**：Integration / Webhook Manager
  - **支撑场景**：事件驱动的系统集成
  - **主要输入**：事件订阅配置、Webhook URL、事件类型
  - **主要输出**：事件推送、推送历史、重试日志
  - **约束**：支持订阅平台事件（任务完成、模型发布、下发结果等）；支持事件类型/URL/密钥/重试策略配置；推送失败自动重试3次；保留推送历史记录；Pr = M（Should Have）

- **FUNC-INTG-006 – SDK支持**
  - **终端**：Interface / 文档
  - **所属模块**：Integration / SDK Manager
  - **支撑场景**：开发者友好的API调用
  - **主要输入**：SDK示例代码、API定义
  - **主要输出**：Python SDK、示例代码、版本管理
  - **约束**：提供Python SDK；支持调用推理服务/数据集管理/任务提交；SDK版本与API版本同步迭代；包含示例代码；Pr = M（Should Have）

## 二、身份认证集成

- **FUNC-INTG-010 – 企业IDP集成配置**
  - **终端**：PC / Interface
  - **所属模块**：Auth / IDP Connector
  - **支撑场景**：单点登录与企业身份管理
  - **主要输入**：IDP元数据、属性映射规则
  - **主要输出**：IDP配置、属性映射、多IDP管理
  - **约束**：支持SAML 2.0/OAuth2/LDAP；配置Entity ID/SSO URL/证书；属性映射（用户名/邮箱/部门→SMP字段）；多IDP支持（不同BU对接不同IDP）；Pr = H（Must Have）

- **FUNC-INTG-011 – SCIM用户同步**
  - **终端**：API / 后端
  - **所属模块**：Auth / SCIM Syncer
  - **支撑场景**：用户生命周期自动化管理
  - **主要输入**：SCIM协议请求、IDP用户变更事件
  - **主要输出**：SMP用户/组织数据、同步状态
  - **约束**：支持SCIM 2.0协议；自动同步用户/组织；IDP侧离职人员自动同步为停用状态；增量+全量同步模式；Pr = M（Should Have）

- **FUNC-INTG-012 – SSO集成测试工具**
  - **终端**：PC / Interface
  - **所属模块**：Auth / SSO Test Tool
  - **支撑场景**：IDP配置验证
  - **主要输入**：测试用户凭证、测试场景
  - **主要输出**：测试结果、属性映射验证
  - **约束**：管理员在配置界面进行SSO流程测试；验证属性映射是否正确；Pr = M（Should Have）

- **FUNC-INTG-013 – SSO故障兜底**
  - **终端**：PC / Interface / Login
  - **所属模块**：Auth / Fallback Handler
  - **支撑场景**：平台可用性保障
  - **主要输入**：本地管理员账号、密码
  - **主要输出**：登录授权、访问令牌
  - **约束**：IDP不可用时，管理员账号支持本地密码登录；普通用户提示IDP故障信息；Pr = H（Must Have）

## 三、第三方推理服务纳管

- **FUNC-INTG-015 – 第三方推理服务注册**
  - **终端**：PC / Interface
  - **所属模块**：Integration / Service Registry
  - **支撑场景**：异构推理服务统一管理
  - **主要输入**：服务URL、协议类型、认证信息、超时配置
  - **主要输出**：服务配置ID、纳管状态
  - **约束**：支持HTTP/gRPC/WebSocket协议；填写URL/认证方式/超时；连接测试；纳管后统一监控；Pr = M（Should Have）

- **FUNC-INTG-016 – 推理服务代理**
  - **终端**：API / 后端
  - **所属模块**：Integration / Service Proxy
  - **支撑场景**：透明访问异构推理服务
  - **主要输入**：服务ID、推理请求
  - **主要输出**：推理结果
  - **约束**：通过平台API网关代理第三方服务调用；统一鉴权/限流/日志；调用方无需感知后端差异；Pr = M（Should Have）

- **FUNC-INTG-017 – 推理服务健康检查**
  - **终端**：API / 后端
  - **所属模块**：Integration / Service Health Monitor
  - **支撑场景**：推理服务可用性监控
  - **主要输入**：服务ID、检查间隔
  - **主要输出**：健康状态、故障告警
  - **约束**：定期对纳管的第三方服务进行探活；不可用时告警；服务列表标注状态；Pr = M（Should Have）

- **FUNC-INTG-018 – 推理服务容量规划支持**
  - **终端**：PC / Interface
  - **所属模块**：Integration / Capacity Planner
  - **支撑场景**：推理资源成本优化
  - **主要输入**：时间范围、服务ID
  - **主要输出**：调用量趋势、容量建议
  - **约束**：统计各推理服务调用量趋势；辅助容量规划决策；Pr = L（Could Live Without）

---

## 四、边端集成

- **FUNC-INTG-020 – 边端Agent协议**
  - **终端**：API / 后端
  - **所属模块**：Edge / Agent Protocol
  - **支撑场景**：边端与中心通信基础
  - **主要输入**：Agent注册信息、心跳消息
  - **主要输出**：连接管理、任务分发
  - **约束**：边端服务器通过Agent与平台通信；支持长连接/轮询两种模式；Agent心跳保活机制；Pr = H（Must Have）

- **FUNC-INTG-021 – 模型下发协议**
  - **终端**：API / 后端
  - **所属模块**：Edge / Model Distribution
  - **支撑场景**：模型从中心到边端的完整流程
  - **主要输入**：模型版本、目标边端、下发策略
  - **主要输出**：下发任务、执行结果
  - **约束**：完整流程（模型版本选择→生成下发任务→Agent接收→下载模型文件→验证完整性（MD5/SHA256）→加载服务→回报结果）；Pr = H（Must Have）

- **FUNC-INTG-022 – 边端状态上报**
  - **终端**：API / 后端
  - **所属模块**：Edge / Status Reporter
  - **支撑场景**：边端实时状态可观测性
  - **主要输入**：边端上报数据
  - **主要输出**：状态数据、监控Dashboard
  - **约束**：定期上报在线状态/CPU/内存/磁盘使用率/运行中的模型服务列表/推理调用量；Pr = H（Must Have）

- **FUNC-INTG-023 – 边端离线处理**
  - **终端**：API / 后端
  - **所属模块**：Edge / Offline Handler
  - **支撑场景**：边端连接中断的故障恢复
  - **主要输入**：离线边端ID、下发任务队列
  - **主要输出**：任务重试、故障告警
  - **约束**：边端离线期间的下发任务进入队列；恢复在线后自动重试；离线超过阈值触发告警；Pr = M（Should Have）

---

> **约束**：所有清单仅基于 SOR V1.0 内容抽取；优先级 H=Must Have，M=Should Have，L=Could Live Without。
