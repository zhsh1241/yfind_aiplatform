# 02-01 业务流程清单 – 数据管理域

> **范围**：数据管理域（DATA）包含数据源接入、多模态数据采集与加工、数据标注、数据集生命周期管理、数据增强及数据集门户等完整的数据治理流程。
> **组织方式**：按子流程分组，每个流程含触发条件、主流程步骤、分支/异常处理。

---

## 一、数据源接入流程（DATA-001）

**触发条件**：数据标注工程师或BU管理员需要接入新的数据源

**主要参与角色**：
- 数据标注工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 选择数据源类型 → 支持关系型DB、文件存储、对象存储、流媒体、时序数据、工业协议、API等多种数据源
2. 配置连接参数 → 输入数据源连接信息（地址、端口、用户名、密码等）
3. 连接测试 → 验证连接可用性
4. 设置团队共享权限 → 配置数据访问权限（按BU/项目/用户）
5. 保存激活 → 数据源纳管，可供后续采集使用

**异常流程**：
- 连接失败：检查连接参数配置、网络连通性，重试连接
- 权限不足：申请BU管理员授权后重新配置

**主要输入**：数据源信息、连接参数、权限配置

**主要输出**：已激活的数据源、数据连接凭证

**关联功能**：FUNC-DATA-001, FUNC-DATA-002

---

## 二、多模态数据采集与加工流程（DATA-002）

**触发条件**：需要从数据源采集数据并进行格式标准化加工

**主要参与角色**：
- 数据标注工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 选择数据源 → 从已激活的数据源列表中选择
2. 配置采集规则 → 指定采集数据类型：文本、图片、音频、视频
3. 执行采集 → 触发数据采集任务
4. 进入加工流水线 → 按数据类型分类处理
5. 格式转换 → 统一转换为平台标准格式
6. 内容安全检测 → 识别敏感内容、涉黄、涉暴信息
7. 类型专项加工 → 根据数据类型进行深层次处理（见分支）
8. 生成预处理数据集 → 完成加工并保存

**分支流程**：

- **图片数据加工**：
  - 几何变换（旋转、缩放、翻转等）
  - 色彩处理（白平衡、色彩空间转换）
  - 分辨率标准化

- **音频数据加工**：
  - 降噪处理
  - 格式转换（转换为WAV/MP3等标准格式）
  - 采样率标准化

- **文本数据加工**：
  - 文本清洗（去除特殊字符、HTML标签）
  - 分词处理
  - 多语言支持（语言识别、字符集转换）

- **视频数据加工**：
  - 视频抽帧（按指定间隔抽取关键帧）
  - 视频分段（按时间或内容分割）
  - 分辨率标准化

**异常流程**：
- 采集失败：检查数据源连接、采集规则配置
- 格式转换失败：人工审查并调整转换规则
- 安全检测拦截：标记为敏感内容，由管理员处置

**主要输入**：原始数据源、采集配置、加工规则

**主要输出**：预处理数据集、加工日志、质量报告

**关联功能**：FUNC-DATA-003, FUNC-DATA-004, FUNC-DATA-005

---

## 三、数据标注流程（DATA-003）

**触发条件**：数据预处理完成，进入标注阶段

**主要参与角色**：
- 数据标注工程师
- 数据审核工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 创建标注任务 → 选择标注场景：图片分类、目标检测、语义分割、语音标注、文本标注；**同时选择是否启用数据审核流程**（可选，默认不启用）
2. 配置标签模板 → 定义标签体系、标注规范（**注：标签模板不包含质量标准**）
3. 分配标注人员 → 根据能力匹配分配标注员
4. 执行标注 → 使用在线标注工具或离线预标注辅助方式
5. 提交标注结果 → 标注工程师提交标注结果
6. 审核阶段（**仅当创建任务时选择了审核流程时执行**）→ 数据审核工程师审核（通过/驳回）
   - 通过：质量满足要求，标注完成
   - 驳回：说明原因，要求标注工程师修正
   - 未启用审核：标注提交后直接进入数据集生成
7. 生成标注数据集 → 汇总所有通过审核（或直接提交）的标注结果

**分支流程**：

- **预标注模式**：
  - 调用AI预标注服务进行初始标注
  - 人工标注工程师修正预标注结果
  - 提高标注效率

- **离线标注模式**：
  - 导出未标注数据集
  - 标注工程师本地标注
  - 导入标注结果
  - 支持多地协同标注

**异常流程**：
- 标注质量不合格：驳回并详细说明原因，要求重新标注
- 人员不足：申请追加标注资源或延长完成期限
- 标签体系不清：暂停任务，完善标签定义后重启

**主要输入**：预处理数据集、标注场景定义、标签体系

**主要输出**：标注数据集、标注质量报告、标注统计数据

**关联功能**：FUNC-DATA-006, FUNC-DATA-007, FUNC-DATA-008

---

## 四、原始数据集管理流程（DATA-004）

**触发条件**：完成数据源接入和初步处理，需要将其纳管为可复用的数据集

**主要参与角色**：
- 数据标注工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 创建或导入数据集 → 新建原始数据集记录或直接导入
2. 添加元数据标签 → 标注数据集名称、描述、创建日期、数据所有者、数据分类等
3. 版本发布 → 标记版本号，发布首个版本
4. 权限配置 → 设置访问权限（公开/团队内/个人）
5. 激活使用 → 数据集可被后续流程引用

**主要输入**：原始数据、元数据描述

**主要输出**：已发布的原始数据集、版本记录

**关联功能**：FUNC-DATA-040, FUNC-DATA-043

---

## 五、预处理数据集管理流程（DATA-005）

**触发条件**：从原始数据集进行多模态数据采集与加工完成

**主要参与角色**：
- 数据标注工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 从原始数据集生成 → 关联源数据集
2. 记录加工血缘 → 记录所有加工步骤、参数、时间戳
3. 添加处理元数据 → 标注加工方式、版本、质量指标
4. 版本控制 → 管理预处理数据集的不同版本
5. 权限配置 → 配置数据集访问权限
6. 激活使用 → 预处理数据集可供后续标注或训练使用

**主要输入**：原始数据集、加工配置

**主要输出**：预处理数据集、加工血缘跟踪、版本信息

**关联功能**：FUNC-DATA-041（预处理数据集管理）, FUNC-DATA-010（格式转换引擎）, FUNC-DATA-043（数据血缘追踪）

## 六、标注数据集管理流程（DATA-006）

**触发条件**：数据标注流程完成，需要将标注结果纳管为数据集

**主要参与角色**：
- 数据标注工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 导入标注结果 → 从标注工具导出并导入平台；**或直接导入已标注的外部数据集，平台自动进行格式解析与转换**
2. 关联标注场景 → 链接标注任务和标注场景定义
3. 质量检查 → 验证标注数据完整性和准确性
4. 生成版本 → 创建标注数据集版本
5. 权限配置 → 设置数据集访问权限
6. 发布激活 → 标注数据集可供模型训练使用

**主要输入**：标注任务输出、标注工具导出数据

**主要输出**：标注数据集、质量检查报告、版本信息

**关联功能**：FUNC-DATA-013, FUNC-DATA-014

---

## 七、数据增强流程（DATA-007）

**触发条件**：训练数据规模不足，需要通过数据增强扩展数据集

**主要参与角色**：
- 数据标注工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 选择源数据集 → 从现有数据集中选择待增强的数据
2. 选择增强类型 → 根据数据类型选择增强策略
3. 配置增强参数 → 设置具体的增强参数和强度
4. 自定义增强插件（可选）→ 上传自定义增强算法
5. 执行增强 → 触发数据增强任务
6. 生成增强数据集 → 保存增强结果为新数据集

**分支流程**：

- **图像数据增强**：
  - 几何变换：旋转、缩放、平移、翻转、错切
  - 色彩空间：亮度调整、对比度调整、饱和度调整、色温调整
  - 噪声注入：高斯噪声、椒盐噪声、泊松噪声
  - 遮挡增强：随机遮挡、网格遮挡、混合图片
  - 自动化增强：AutoAugment、RandAugment等自动策略

- **音频数据增强**：
  - 变速处理：加速/减速
  - 变调处理：改变音高
  - 添加背景噪声：模拟真实录音环境
  - 音量调整：随机改变音量

- **文本数据增强**：
  - 同义词替换：使用同义词替换原词
  - 回译增强：多语言回译
  - 随机插入/删除/交换
  - 句子改写

**异常流程**：
- 增强参数不合理：调整参数后重新执行
- 增强后数据质量下降：调整增强策略或手动检查

**主要输入**：源数据集、增强策略、增强参数

**主要输出**：增强数据集、增强统计报告

**关联功能**：FUNC-DATA-015, FUNC-DATA-016

---

## 八、数据集门户申请流程（DATA-008）

**触发条件**：用户需要查找和申请使用数据集

**主要参与角色**：
- 数据标注工程师
- 数据审核工程师
- 模型训练工程师
- 模型应用工程师
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 浏览数据集门户 → 访问数据集门户首页
2. 搜索和筛选 → 按照访问量、存储量、数据类型、创建时间等维度筛选
3. 查看数据集详情 → 查阅数据集描述、元数据、样例数据
4. 申请使用 → 填写使用申请表（申请理由、使用期限、使用场景）
5. 管理员审批 → 数据集所有者或BU管理员审批申请
   - 同意：配置权限，用户获得访问权限
   - 拒绝：返回拒绝原因
6. 获得访问权限 → 用户可下载或在线访问数据集

**分支流程**：
- **公开数据集**：无需审批，直接获得访问权限
- **受限数据集**：需要审批流程

**异常流程**：
- 申请被拒绝：查看拒绝原因，补充材料后重新申请
- 访问权限过期：重新申请延期

**主要输入**：搜索条件、申请信息

**主要输出**：数据集访问权限、下载链接

**关联功能**：FUNC-DATA-017, FUNC-DATA-018
