# 02-04 业务流程清单 – 平台运营与资源管理域

> **范围**：平台运营与资源管理域包括系统管理（SYS）和计算资源管理（RES）的所有流程，涵盖平台初始化、组织架构配置、用户权限管理、数据安全管控、计算资源管理、存储管理和镜像管理等内容。
> **组织方式**：按子流程分组，每个流程含触发条件、主流程步骤、分支/异常处理。

---

## 一、平台初始化与组织架构配置流程（SYS-001）

**触发条件**：平台部署完成，需进行初期配置和组织设置

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 配置SSO与身份认证系统集成 → 
   - 选择身份认证方式：
     - LDAP/AD集成：对接企业目录服务
     - OAuth 2.0：支持第三方账号登录
     - SAML 2.0：企业级单点登录
     - 本地账号：仅在SSO不可用时使用
   - 配置认证服务器地址和凭证
   - 测试认证流程可用性
   - 配置会话过期时间和令牌刷新策略

2. 创建集团级组织架构树 → 
   - **顶层：集团级**
     - 配置集团名称
     - 定义集团管理员
   - **中层：业务单元（BU）**
     - 创建各BU（如研发BU、制造BU、运营BU等）
     - 为每个BU指定BU管理员
     - 配置BU级别的数据隔离策略
   - **底层：项目层**
     - 在BU下创建具体项目
     - 配置项目所有者和成员
   - 可视化展示组织结构树，支持动态调整

3. 定义全局角色权限策略 → 
   - **RBAC（基于角色的访问控制）**：
     - 定义全局角色：超级管理员、BU管理员、项目管理员、数据工程师、模型训练工程师、推理工程师、普通用户等
     - 为每个角色定义权限集合
     - 权限粒度：资源级、操作级
   - **ABAC（基于属性的访问控制）**：
     - 定义用户属性：部门、岗位、级别、成本中心等
     - 定义资源属性：数据分级、项目归属、敏感度等
     - 定义访问规则：用户属性+资源属性 → 操作权限
   - 遵循最小权限原则：用户仅获得完成工作所需的最少权限

4. 配置公共基础资源 → 
   - **数据集**：上传企业级通用数据集（行业规范数据、基础语料库）
   - **模型**：发布平台级预训练模型
   - **镜像**：配置基础开发环境镜像（PyTorch、TensorFlow等）
   - **计算资源配额**：配置平台级资源总额

5. 配置全局数据存储标准与生命周期策略 → 
   - **数据分级标准**：
     - 公开数据：可对外分享
     - 内部数据：仅企业内部使用
     - 受限数据：需特殊批准才能访问
     - 机密数据：严格访问控制
   - **生命周期策略**：
     - 创建期：数据采集和预处理
     - 活跃期：数据被频繁使用
     - 冷却期：数据使用频率降低，迁移到低成本存储
     - 归档期：长期保存备份
     - 销毁期：根据法规要求安全删除数据
   - **默认存储位置**：配置不同数据类型的默认存储路径

6. 授权BU子管理员 → 
   - 为每个BU管理员分配权限
   - 定义BU管理员可以管理的资源范围
   - 配置审批流程：BU管理员级别的操作需求经谁审批

**异常流程**：
- SSO连接失败：启用本地认证临时方案，同时联系IT部门排查SSO服务
- 组织架构设计有误：支持重新调整，但需进行权限重新配置

**主要输入**：认证系统信息、组织结构定义、角色权限模板、资源初始化清单

**主要输出**：已配置的认证系统、组织结构树、角色权限配置、基础资源

**关联功能**：FUNC-SYS-001, FUNC-SYS-002, FUNC-SYS-003

## 二、BU管理员日常管理流程（SYS-001b）

**触发条件**：BU日常运营需要进行用户和资源管理

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 管理BU内用户清单 → 
   - **添加用户**：从AD/LDAP导入或手动添加，分配初始角色
   - **删除用户**：离职或转岗用户的权限撤销和数据清理
   - **修改用户**：调整角色、权限、联系方式等信息
   - **批量操作**：支持通过Excel导入进行批量操作

2. 配置BU级角色 → 
   - 基于企业级角色进一步细化BU级自定义角色
   - 配置这些角色的权限范围
   - 将BU内用户分配给不同角色

3. 管理BU数据集/模型/镜像 → 
   - **BU级数据集**：发布团队内通用数据集
   - **BU级模型**：共享调优完成的基础模型
   - **BU级镜像**：维护BU标准开发环境镜像（预装特定库）
   - 设置这些资源的使用权限和版本管理

4. 向总部申请资源支持 → 
   - 评估BU当前资源使用情况
   - 提交资源扩容申请（CPU、GPU、存储配额）
   - 说明业务需求和时间紧急性
   - 跟踪审批进展

**主要输入**：用户信息、角色定义、资源清单

**主要输出**：更新的用户清单、角色配置、资源申请记录

**关联功能**：FUNC-SYS-004, FUNC-SYS-005

---

## 三、用户权限管理流程（SYS-002）

**触发条件**：新用户入职、用户岗位变更、离职等事件触发权限变更

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 新建用户 → 
   - **自动导入**：对接SSO系统，用户首次登录时自动创建账户
   - **手动创建**：管理员在平台中直接创建本地账户
   - 记录用户基本信息：姓名、工号、部门、邮箱等

2. 分配角色 → 
   - 根据RBAC+ABAC策略为用户分配初始角色
   - 参考职位、部门、技能水平等因素
   - 记录权限分配理由和审批者

3. 遵循最小权限原则 → 
   - 用户仅获得完成其工作角色所需的权限
   - 定期审查已授予权限，撤销不必要权限
   - 避免权限过度分配

4. 操作审计记录 → 
   - 记录所有权限修改操作（谁、何时、修改了什么）
   - 保留完整的权限变更历史
   - 支持权限变更回溯和撤销

5. 定期权限复审 → 
   - **频率**：至少每半年进行一次
   - **流程**：
     - 管理员检查用户当前权限
     - 验证权限与当前岗位的匹配度
     - 撤销冗余权限
     - 获取用户和主管的确认
   - **记录**：保留复审记录供审计查看

**异常流程**：
- 用户离职：立即撤销所有权限，禁用账户，清理用户数据访问权限
- 岗位变更：撤销旧岗位权限，分配新岗位权限
- 权限争议：通过审计日志追溯权限分配理由，由上级管理员仲裁

**主要输入**：用户信息、角色权限模板、审批意见

**主要输出**：用户权限配置、权限审计日志

**关联功能**：FUNC-SYS-006, FUNC-SYS-007

## 四、数据安全管控流程（SYS-003）

**触发条件**：平台日常运营过程中的数据安全保护

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 数据分级分类 → 
   - **数据分级**（按敏感度）：
     - L1 公开：可对外公开分享的数据
     - L2 内部：仅企业内部使用
     - L3 受限：涉及商业机密或个人隐私
     - L4 机密：涉及国家安全或核心知识产权
   - **数据分类**（按业务）：
     - 客户数据
     - 产品数据
     - 运营数据
     - 财务数据
   - 为每个数据集标注分级和分类标签

2. 内容安全审查 → 
   - **敏感信息识别**：
     - 个人隐私信息（身份证、手机号、邮箱）
     - 商业机密（价格、策略、竞争信息）
     - 涉及国家安全的信息
   - **内容审核**：
     - 涉黄内容识别
     - 涉暴力和恐怖主义内容识别
     - 违法违规内容识别
   - **审核方式**：
     - 自动审核：使用内容识别模型
     - 人工审核：对自动标记的内容人工确认
   - 对不合规内容进行处理（删除、脱敏、隔离）

3. 存储加密配置 → 
   - **静态数据加密**：
     - 对所有受限和机密数据启用加密存储
     - 使用业界标准加密算法（AES-256）
     - 配置密钥管理系统（KMS）
     - 定期轮换加密密钥
   - **备份加密**：备份数据也需加密存储

4. 传输安全配置 → 
   - **传输协议**：
     - 所有网络传输使用HTTPS/TLS加密
     - 配置TLS版本和加密套件
   - **VPN连接**：受限内网访问需使用VPN
   - **证书管理**：定期检查和更新SSL证书

5. 操作日志安全存储 → 
   - **日志收集**：
     - 收集所有用户操作日志（谁、做了什么、何时）
     - 收集系统事件日志
     - 收集数据访问日志
   - **日志存储**：
     - 日志进行加密存储
     - 配置日志保留期（至少1年）
     - 实现日志不可篡改（使用不可变存储或区块链技术）
   - **日志监控**：
     - 实时检测异常操作（大量数据访问、权限提升等）
     - 触发告警并通知安全团队

6. 定期安全审计 → 
   - **审计频率**：至少每季度进行一次
   - **审计内容**：
     - 审查用户权限与岗位匹配度
     - 检查数据访问日志是否有异常
     - 验证加密和安全配置是否生效
     - 检查是否有未授权数据访问
     - 检查数据分级分类是否完整
   - **审计报告**：生成审计报告，提出改进建议
   - **持续改进**：根据审计结果持续完善安全管控

**异常流程**：
- 发现敏感信息泄露：立即隔离相关账户，开展调查，通知相关方
- 检测到非授权访问：撤销权限，开展取证，更新安全策略

**主要输入**：数据分类标准、审核规则、加密配置

**主要输出**：数据分级分类清单、安全审计报告、加密配置

**关联功能**：FUNC-SYS-008, FUNC-SYS-009, FUNC-SYS-010

## 五、计算资源管理流程（RES-001）

**触发条件**：平台日常资源调度和管理

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

### 阶段一：集群管理

1. 纳管集群节点 → 
   - 向平台添加计算节点（物理机或虚拟机）
   - 配置节点基本信息：IP、主机名、操作系统、硬件配置
   - 安装平台agent和容器运行时（Docker/containerd）
   - 验证节点可用性

2. 查看资源总量与分配 → 
   - 展示所有节点的CPU、内存、存储容量
   - 显示已分配资源和可用资源
   - 展示资源使用趋势（时间序列图表）

3. 配置调度策略 → 
   - **资源隔离**：
     - 配置不同用户/团队的资源隔离
     - 支持独占资源池（确保资源可用性）
     - 支持共享资源池（提高利用率）
   - **调度算法**：
     - 支持最少负载均衡
     - 支持亲和性调度（优先在特定节点运行）
     - 支持反亲和性调度（分散到不同节点）
   - **优先级**：
     - 定义任务优先级
     - 高优先级任务可抢占低优先级任务资源

### 阶段二：资源池管理

4. 创建资源池 → 
   - 给资源池起名称和描述
   - 选择资源池模式：
     - **独占模式**：资源池内节点仅供指定用户/团队使用
     - **共享模式**：资源池内节点由多个用户/团队共享
   - 配置所有者和管理员

5. 配置节点分配与配额 → 
   - 将集群中的节点分配给资源池
   - 为资源池设置资源配额：
     - CPU总额
     - 内存总额
     - GPU/NPU数量
   - 配置单个任务的最大资源限制
   - 配置资源队列和优先级

6. 实时监控资源使用 → 
   - 显示资源池整体使用情况
   - 显示各用户/团队的资源使用占比
   - 显示节点级别的实时监控数据
   - 设置资源告警阈值（如使用率>80%时告警）

### 阶段三：AI芯片管理

7. 注册AI芯片 → 
   - 识别节点上的AI芯片（GPU/NPU）
   - 记录芯片信息：
     - 芯片型号（如RTX 3090、华为昇腾310等）
     - 显存大小
     - 硬件版本
     - 驱动版本

8. 配置调度策略 → 
   - **芯片分配策略**：
     - 独占分配：一个任务独占一个GPU
     - 共享分配：多个任务共享一个GPU（需要容器化支持）
   - **优先级调度**：
     - 定义不同任务类型对芯片的优先级
   - **备用芯片**：配置故障自动转移到备用芯片

9. 设置展示名称 → 
   - 为每个芯片配置友好的显示名称
   - 用户在任务配置时可选择具体芯片型号

**异常流程**：
- 节点故障：自动隔离故障节点，迁移其上的任务到其他节点
- 资源不足：任务进入等待队列，当资源释放时自动调度
- GPU驱动异常：系统检测并告警，人工处理更新

**主要输入**：节点信息、资源配额配置、调度策略

**主要输出**：资源配置清单、监控报告、调度日志

**关联功能**：FUNC-RES-001, FUNC-RES-002, FUNC-RES-003

## 六、存储中心管理流程（RES-002）

**触发条件**：平台日常存储资源管理

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 接入存储池 → 
   - 配置存储后端（本地存储、NAS、分布式存储如Ceph等）
   - 验证存储连接可用性
   - 配置存储性能参数（IOPS、吞吐量）
   - 配置备份策略

2. 元数据管理 → 
   - 记录每个存储池的基本信息
   - 配置访问权限和隔离策略
   - 设置存储配额和告警阈值

3. 查看各团队存储分配 → 
   - 按BU/团队/项目维度展示存储使用情况
   - 显示存储使用趋势
   - 识别存储占用大的数据集

4. 冷热数据迁移 → 
   - **配置迁移策略**：
     - 定义冷数据标准：N天内未访问的数据视为冷数据
     - 选择迁移目标存储（如高成本SSD迁移到低成本HDD）
   - **执行迁移**：
     - 按照策略自动将冷数据迁移到低成本存储
     - 支持手动触发迁移
   - **查看迁移记录**：
     - 显示迁移历史
     - 显示每次迁移的数据量和耗时

**异常流程**：
- 存储告急：通知团队清理无用数据，或申请扩容
- 存储故障：触发备份恢复流程，确保数据可用

**主要输入**：存储池配置、迁移策略

**主要输出**：存储使用报告、迁移记录

**关联功能**：FUNC-RES-004, FUNC-RES-005

## 七、镜像管理流程（RES-003）

**触发条件**：开发人员需要构建和维护标准化的开发环境镜像

**主要参与角色**：
- BU级子管理员
- 超级管理员

**主流程步骤**：

1. 导入镜像 → 
   - **导入方式**：
     - 本地文件：上传预建的镜像文件（tar格式）
     - Dockerfile：上传Dockerfile，平台自动构建镜像
     - 代码包：上传源代码包，根据配置文件自动构建
   - 验证镜像完整性和可用性

2. 配置架构 → 
   - 指定镜像支持的处理器架构：
     - ARM（用于移动设备和边端设备）
     - x86（用于服务器）
   - 支持多架构镜像（同一个镜像支持多种架构）

3. 版本管理 → 
   - **编辑元数据**：
     - 镜像名称、描述
     - 版本号（遵循语义化版本2.0）
     - 创建时间、维护者信息
   - **标签管理**：
     - 为镜像添加标签（如pytorch-1.9、gpu-enabled）
     - 便于用户快速找到所需镜像
   - **描述更新**：
     - 记录镜像包含的软件和版本
     - 记录已知问题和限制
     - 记录使用说明

4. 配置仓库信息 → 
   - 配置镜像仓库地址（内部镜像仓库或Docker Hub等）
   - 配置仓库访问凭证
   - 配置镜像拉取加速（如使用国内镜像源）

5. 支持模型镜像分离 → 
   - **基础镜像**：包含框架、依赖库、开发工具（如pytorch:1.9-cuda11）
   - **模型镜像**：在基础镜像之上添加具体的预训练模型
   - 分离优势：
     - 减小镜像体积
     - 便于模型独立更新
     - 支持镜像复用

6. 版本生命周期管理 → 
   - **标记版本状态**：
     - 开发版：正在开发中
     - 测试版：供测试使用
     - 稳定版：生产就绪
     - 废弃版：不再维护
   - **自动清理**：配置废弃版本的自动删除策略

7. 废弃/删除镜像版本 → 
   - 标记版本为废弃
   - 通知使用该镜像的用户
   - 在保留期后自动删除（可配置保留期）
   - 保留删除日志供审计

**分支流程**：

- **从容器保存镜像**：
  - 运行容器进行开发和定制
  - 容器配置完成后，保存为新镜像
  - 注册新镜像到镜像仓库

- **从CI/CD流水线构建镜像**：
  - 代码提交自动触发镜像构建
  - 构建成功后自动推送到仓库
  - 构建失败时通知开发者

**异常流程**：
- 镜像构建失败：查看构建日志，修复Dockerfile后重试
- 镜像拉取失败：检查网络连接和仓库凭证
- 镜像体积过大：优化Dockerfile，使用多阶段构建减小体积

**主要输入**：镜像文件/Dockerfile/代码包、元数据配置

**主要输出**：已发布的镜像、版本记录、镜像仓库链接

**关联功能**：FUNC-RES-006, FUNC-RES-007, FUNC-RES-008
