# 07 工作台与调度中心功能规格

> **范围**：平台首页工作台（Dashboard）和调度中心（Scheduling Center）的 UI 功能规格
> **组织方式**：按模块分组，每功能点含终端、模块路径、支撑场景、输入输出、约束

---

## 一、首页工作台 Dashboard

### 功能概述
工作台是用户登录后的默认落地页，提供个人化的任务入口、实例状态汇总和平台动态。

- **FUNC-DASH-001 – 5步流程引导卡**
  - 说明：对新用户展示平台使用流程引导（接入数据→标注数据→训练模型→评估模型→部署推理），每步有快捷入口按钮
  - 约束：首次登录后 14 天内展示；用户可手动关闭；完成某步骤后对应步骤自动标绿，Pr = M（Should Have）

- **FUNC-DASH-002 – 实例统计矩阵**
  - 说明：5种实例类型（数据加工/标注任务/训练任务/推理服务/开发任务）×3种状态（运行中/已完成/失败）的统计矩阵，点击单元格跳转到对应列表（带预置过滤条件）
  - 约束：数据实时刷新（30秒轮询）；仅显示当前租户/BU 内数据，Pr = H（Must Have）

- **FUNC-DASH-003 – 数据集分类统计**
  - 说明：当前租户数据集按类型（文本/图片/音频/视频/多模态）的数量和存储用量统计，环形图展示
  - 约束：数量和存储用量分别展示；点击类型跳转数据集门户并预置类型筛选，Pr = M（Should Have）

- **FUNC-DASH-004 – 个人信息与快捷操作面板**
  - 说明：展示当前用户头像、角色、所属BU/项目；提供最近访问的5个资源快捷链接；通知消息角标
  - 约束：最近访问列表按访问时间降序，最多5条；通知角标数量上限99，Pr = M（Should Have）

- **FUNC-DASH-005 – 资源用量概览**
  - 说明：展示当前租户 GPU/NPU/存储配额使用情况（已用/总量/百分比）；超过 80% 时变色警示
  - 约束：数据从资源域配额管理模块拉取；超过 90% 发送站内通知，Pr = H（Must Have）

## 二、调度中心实例总览

### 功能概述
调度中心提供平台所有训练/推理/数据处理任务的统一调度视图，含实时状态、历史趋势和问题诊断。

- **FUNC-SCHED-001 – 实例状态分布卡**
  - 说明：以状态卡片形式展示平台当前 Pending/Running/Completed/Failed 任务总数，点击跳转过滤列表
  - 约束：状态数量实时刷新（10秒轮询）；区分训练任务/推理服务/数据处理任务三类，Pr = H（Must Have）

- **FUNC-SCHED-002 – 任务趋势折线图**
  - 说明：展示近 12h/1天/7天/30天/60天新提交任务量和完成率趋势折线图，支持时间范围切换
  - 约束：时间粒度随范围自动调整（12h=15min粒度，60天=1天粒度）；图表支持导出为 PNG/CSV，Pr = M（Should Have）

- **FUNC-SCHED-003 – 失败实例排行**
  - 说明：展示近 24h/7天 失败次数最多的 Top10 任务（含任务名/所属BU/失败次数/最近失败时间）；点击查看失败日志
  - 约束：超级管理员可见全租户数据；BU Admin 仅见本 BU 数据，Pr = M（Should Have）

- **FUNC-SCHED-004 – 调度队列可视化**
  - 说明：展示当前 Pending 队列中的任务列表（按优先级排序），含等待时间、资源需求、提交人
  - 约束：仅对超级管理员和 BU Admin 可见；支持手动提升/降低任务优先级（拖拽排序），Pr = M（Should Have）

- **FUNC-SCHED-005 – 智能调度助手**
  - 说明：AI 驱动的调度优化建议，分析当前队列和资源利用率，推荐调度策略调整（如"GPU空闲率高，建议降低最小资源预留"）
  - 约束：建议为只读（不自动执行）；附操作按钮"采纳建议"触发配置变更，Pr = L（Could Have）

---

> **约束**：所有清单仅基于 SOR V1.0 及截图分析内容；优先级 H=Must Have，M=Should Have，L=Could Live Without。
