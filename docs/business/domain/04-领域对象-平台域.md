# 04 领域对象 – 平台域（Platform Domain）

> **范围**：SMP平台的横切基础设施域，涵盖多租户组织管理、用户认证与权限、操作审计日志、消息通知等核心平台能力。
> **组织方式**：按聚合根分组，每个领域对象含标识、核心属性、状态机、关联关系和操作清单。
> **对应功能文档**：`05-01-系统功能-平台基础.md` / `05-02-系统功能-国际化与审计日志.md` / `05-03-系统功能-通知与文件管理.md`

## 聚合根1：租户（Tenant Aggregate）

**领域对象**：`Tenant`  
**标识**：`tenantId: UUID`  

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `tenantId` | UUID | 租户唯一标识 |
| `name` | String | 租户/组织名称 |
| `tenantType` | Enum | 租户类型：`CORP`（集团）/ `BU`（事业部）/ `PROJECT`（项目） |
| `parentId` | UUID | 父租户ID（支持树形结构，集团为null） |
| `timezone` | String | 时区设置（如"Asia/Shanghai"） |
| `defaultLocale` | Enum | 默认语言：`zh-CN`（中文）/ `en-US`（英文）/ `ko-KR`（韩语）/ `ja-JP`（日语） |
| `status` | Enum | 状态（见下方状态机） |
| `config` | Object | BU级覆盖配置（JSON格式，存储该租户的特殊配置） |
| `createdAt` | Timestamp | 创建时间 |
| `updatedAt` | Timestamp | 更新时间 |

**状态机**：
```
活跃 → 禁用 → 已注销
       ↑
       └── 禁用 (可重新激活)
```

- **活跃**：租户正常运营，用户可访问
- **禁用**：租户被管理员禁用，所有用户强制下线，子节点级联禁用
- **已注销**：租户已注销，数据逻辑删除（保留审计记录）

**关联关系**：
- `1:N → User`：一个租户包含多个用户
- `1:N → Tenant`：一个租户可有多个子租户（树形结构）
- `1:N → ResourcePool`：租户可分配获得多个资源池
- `1:N → Role`：租户可定义多个自定义角色

**操作**：
- `createTenant(name, tenantType, parentId)`：创建组织节点
- `updateConfig(configJson)`：更新配置
- `disableTenant()`：禁用租户（级联禁用子节点和用户）
- `enableTenant()`：重新激活租户
- `getTenantTree()`：查看组织树
- `queryChildTenants()`：查询子租户列表

## 聚合根2：用户（User Aggregate）

**领域对象**：`User`  
**标识**：`userId: UUID`  

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `userId` | UUID | 用户唯一标识 |
| `employeeId` | String | 工号（唯一约束，用于员工身份识别） |
| `name` | String | 用户名称 |
| `email` | String | 邮箱地址（唯一） |
| `phone` | String | 手机号码（可选） |
| `tenantIds` | List<UUID> | 绑定的租户列表（支持多BU关联） |
| `status` | Enum | 用户状态：`ACTIVE`（活跃）/ `DISABLED`（禁用）/ `LOCKED`（账号锁定）/ `INACTIVE`（待激活） |
| `authType` | Enum | 认证方式：`LOCAL`（本地账号）/ `SSO`（单点登录） |
| `lastLoginAt` | Timestamp | 最后登录时间 |
| `passwordHash` | String | 密码哈希值（SHA-256 + Salt） |
| `failedLoginCount` | Integer | 连续登录失败次数（用于账号锁定判断） |
| `lockedUntil` | Timestamp | 账号锁定解除时间（5次失败后自动锁定30分钟或管理员解锁） |
| `preferredLocale` | Enum | 用户偏好语言：`zh-CN` / `en-US` / `ko-KR` / `ja-JP` |
| `createdAt` | Timestamp | 创建时间 |
| `updatedAt` | Timestamp | 更新时间 |

**子对象**：`UserTenantRole`

| 属性 | 类型 | 说明 |
|------|------|------|
| `userTenantRoleId` | UUID | 关联记录ID |
| `userId` | UUID | 用户ID |
| `tenantId` | UUID | 租户ID |
| `roleIds` | List<UUID> | 该租户下的角色ID列表 |
| `expiredAt` | Timestamp | 权限过期时间（支持临时权限，为null时表示永久） |
| `grantedAt` | Timestamp | 授权时间 |

**状态机**：
```
待激活 → 活跃 → 锁定 → 活跃（解锁或冷却期到期）
         ↓
       禁用 → 已注销
```

- **待激活**：用户已创建，未激活账户
- **活跃**：用户可正常登录和使用系统
- **锁定**：连续登录失败达5次自动锁定，或管理员手动锁定
- **禁用**：被管理员禁用，强制下线，无法登录
- **已注销**：用户注销，逻辑删除

**关联关系**：
- `N:M → Role`：通过UserTenantRole管理用户-角色关系
- `N:1 → Tenant`：用户绑定到多个租户
- `1:N ← AuditLog`：用户是操作审计日志的操作人
- `1:N ← Notification`：用户是通知消息的接收人

**操作**：
- `createUser(employeeId, name, email)`：创建用户
- `assignRole(tenantId, roleIds, expiredAt)`：分配角色（支持临时权限）
- `disableUser()`：停用用户（触发强制下线）
- `unlockAccount()`：解锁账号
- `resetPassword(newPasswordHash)`：重置密码
- `queryUsers(tenantId, filters)`：查询用户列表
- `changePreferredLocale(locale)`：修改语言偏好
- `logout()`：注销登录

## 聚合根3：角色（Role Aggregate）

**领域对象**：`Role`  
**标识**：`roleId: UUID`  

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `roleId` | UUID | 角色唯一标识 |
| `code` | String | 角色编码（唯一，如SUPER_ADMIN, BU_ADMIN） |
| `name` | String | 角色名称 |
| `scope` | Enum | 角色作用域：`GLOBAL`（全局/预设）/ `TENANT`（租户级自定义）/ `PROJECT`（项目级自定义） |
| `isPreset` | Boolean | 是否内置预设角色（预设角色不可修改，仅可关联） |
| `parentRoleId` | UUID | 继承自的父角色ID（自定义角色可继承预设角色权限上限） |
| `permissions` | List<Permission> | 权限列表 |
| `tenantId` | UUID | 自定义角色所属租户ID（预设角色为null） |
| `status` | Enum | 角色状态：`ACTIVE`（活跃）/ `DEPRECATED`（废弃） |
| `description` | String | 角色描述 |
| `createdAt` | Timestamp | 创建时间 |
| `updatedAt` | Timestamp | 更新时间 |

**内置预设角色**：

| 角色编码 | 角色名称 | 主要权限范围 | 应用场景 |
|---------|---------|-----------|---------|
| `SUPER_ADMIN` | 超级管理员 | 全平台所有权限，包括系统配置、租户管理、用户权限、审计查看 | 平台运维、系统管理员 |
| `BU_ADMIN` | BU子管理员 | BU级管理员，可管理BU下的用户、项目、资源池、角色 | 业务部门管理员 |
| `DATA_ANNOTATOR` | 数据标注工程师 | 标注任务的查看与标注提交、数据集查看 | 数据标注团队成员 |
| `DATA_REVIEWER` | 审核工程师 | 标注任务的审核、驳回、批准，数据集管理 | 数据质量审核人员 |
| `MODEL_TRAINER` | 模型训练工程师 | 模型训练、微调、工程化任务的创建与管理，开发环境使用 | 算法工程师、数据科学家 |
| `MODEL_OPS` | 模型应用工程师 | 模型部署、推理应用的管理，资源调度、监控 | 模型运维、应用工程师 |

**子对象**：`Permission`

| 属性 | 类型 | 说明 |
|------|------|------|
| `permissionId` | UUID | 权限ID |
| `resource` | String | 资源类型（如"Dataset", "Model", "User", "AuditLog"） |
| `action` | String | 操作类型（如"CREATE", "READ", "UPDATE", "DELETE", "EXPORT"） |
| `conditions` | Object | ABAC条件（基于属性的访问控制，可为空表示无条件） |
| `level` | Integer | 权限等级（1=菜单级, 2=功能级, 3=数据行级） |

**状态机**：
```
活跃 → 废弃
```

- **活跃**：角色正常使用，可分配给用户
- **废弃**：角色已废弃（预设角色不可废弃，仅内置角色不废弃）

**约束**：
- 最小权限原则：新用户默认零权限
- 自定义角色权限不可超越父角色权限上限（权限继承受限）
- 内置预设角色不可删除或修改权限，仅可关联

**关联关系**：
- `M:N ← User`：通过UserTenantRole与用户关联
- `M:1 → Tenant`：自定义角色属于某个租户

**操作**：
- `definePresetRole(code, name, permissions)`：定义预设角色（仅系统管理员）
- `createCustomRole(name, tenantId, parentRoleId)`：创建自定义角色
- `assignPermissions(permissionList)`：分配权限
- `deprecateRole()`：废弃角色
- `cloneFromParent(parentRoleId)`：从父角色克隆权限
- `validatePermissionHierarchy()`：验证权限层级约束

## 聚合根4：操作审计日志（AuditLog Aggregate）

**领域对象**：`AuditLog`  
**标识**：`logId: UUID`  

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `logId` | UUID | 日志唯一标识 |
| `eventId` | String | 事件ID（用于链接相关操作） |
| `timestamp` | Timestamp | 操作时间戳（精确到毫秒） |
| `operatorId` | UUID | 操作人ID |
| `operatorName` | String | 操作人名称 |
| `operatorBU` | String | 操作人所属BU |
| `operatorRole` | String | 操作人角色 |
| `action` | Enum | 操作类型枚举（如CREATE, UPDATE, DELETE, EXPORT, APPROVE, REJECT, DOWNLOAD, LOGIN, LOGOUT等） |
| `resourceType` | String | 资源类型（如Dataset, Model, User, Role, AuditLog等） |
| `resourceId` | UUID | 资源ID |
| `resourceName` | String | 资源名称 |
| `tenantId` | UUID | 操作所属租户ID |
| `requestIP` | String | 请求来源IP |
| `userAgent` | String | 浏览器User-Agent |
| `result` | Enum | 操作结果：`SUCCESS`（成功）/ `FAILURE`（失败） |
| `detail` | Object | 详细内容（JSON格式，含before/after字段，记录操作前后状态） |
| `riskLevel` | Enum | 风险等级：`INFO`（信息）/ `WARNING`（警告）/ `CRITICAL`（严重） |
| `signature` | String | SHA-256签名（用于完整性校验） |

**高危操作分类（CRITICAL级别）**：
- 删除数据集、模型等核心业务对象
- 模型下发边端、推理部署
- 修改全局配置、系统参数
- 跨租户操作、权限边界突破
- 批量权限变更、用户禁用
- 导出敏感数据（PII、模型等）

**不可变性约束**：
- 写入后不可修改/删除；支持逻辑标记（如标记为已审核）
- SHA-256签名校验，防篡改
- 独立存储，与业务库物理分离（冷存储）
- 数据保留周期：至少3年

**关联关系**：
- `N:1 → User`：操作的发起人
- `N:1 → Tenant`：操作所属的租户

**操作**：
- `recordOperation(operatorId, action, resourceType, resourceId, detail)`：记录操作（内部调用，禁止外部直接写入）
- `queryAuditLogs(filters, pagination)`：查询审计日志（支持过滤、分页）
- `exportAuditReport(startTime, endTime, format)`：生成安全审计报告（CSV/PDF）
- `verifySignature(logId)`：验证日志签名完整性
- `queryByEvent(eventId)`：按事件ID查询相关日志

## 聚合根5：通知消息（Notification Aggregate）

**领域对象**：`Notification`  
**标识**：`notificationId: UUID`  

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `notificationId` | UUID | 通知唯一标识 |
| `type` | Enum | 通知类型：`TASK_DONE`（任务完成）/ `TASK_FAILED`（任务失败）/ `APPROVAL_REQUEST`（审批请求）/ `APPROVAL_RESULT`（审批结果）/ `RESOURCE_ALERT`（资源告警）/ `MODEL_DEPLOY`（模型部署）/ `SYSTEM_ANNOUNCE`（系统公告）/ `CUSTOM`（自定义） |
| `title` | String | 通知标题 |
| `body` | String | 通知内容 |
| `priority` | Enum | 优先级：`HIGH`（高）/ `MEDIUM`（中）/ `LOW`（低） |
| `senderId` | UUID | 发送人ID（系统消息为null） |
| `recipientId` | UUID | 接收人ID |
| `channels` | List<Enum> | 投递渠道列表：`IN_APP`（应用内）/ `EMAIL`（邮件）/ `IM`（即时通讯） |
| `status` | Enum | 消息状态（见下方状态机） |
| `relatedResourceType` | String | 关联资源类型（如Dataset, Model, Task） |
| `relatedResourceId` | UUID | 关联资源ID |
| `tenantId` | UUID | 所属租户ID |
| `createdAt` | Timestamp | 创建时间 |
| `readAt` | Timestamp | 用户已读时间 |
| `expiredAt` | Timestamp | 消息过期时间 |
| `retryCount` | Integer | 投递重试次数（最多3次） |

**状态机**：
```
PENDING → SENT → READ
           ↓
        FAILED (最多重试3次)
```

- **PENDING**：消息已创建，待发送
- **SENT**：消息已成功投递到至少一个渠道
- **READ**：用户已在应用内查看消息
- **FAILED**：消息投递失败，已超过重试次数（3次），不再重试

**关联关系**：
- `N:1 → User`：通知的接收人
- `1:1 → Resource`：可关联到具体的业务资源（可选）

**操作**：
- `createNotification(type, title, body, recipientId, channels)`：创建通知
- `sendNotifications(notificationIds, channelList)`：批量发送通知
- `markAsRead(notificationId)`：标记单个通知已读
- `markMultipleAsRead(notificationIds)`：批量标记已读
- `deleteExpiredNotifications(expiredBefore)`：删除过期通知
- `retryFailedNotifications(maxRetries)`：重试失败的消息投递
- `queryNotifications(recipientId, filters)`：查询用户通知列表

## 跨聚合根关系总结

| 源聚合根 | 关联 | 目标聚合根 | 说明 |
|---------|------|-----------|------|
| Tenant | 1:N | User | 租户包含多个用户 |
| Tenant | 1:N | Tenant | 租户树形结构（父-子） |
| Tenant | 1:N | ResourcePool | 租户分配获得资源池 |
| Tenant | 1:N | Role | 租户定义自定义角色 |
| User | M:N | Role | 通过UserTenantRole关联 |
| User | N:1 | Tenant | 用户绑定多个租户 |
| User | 1:N ← | AuditLog | 用户发起的操作记录 |
| User | 1:N ← | Notification | 用户接收的通知消息 |
| Role | M:1 | Tenant | 自定义角色属于租户 |
| Role | 1:1 → | Role | 角色继承关系（自定义 → 预设） |
| AuditLog | N:1 | User | 操作人关联 |
| AuditLog | N:1 | Tenant | 操作所属租户 |
| Notification | N:1 | User | 接收人关联 |
| Notification | N:1 | Tenant | 通知所属租户 |

## 核心设计原则

### 1. 多租户隔离
- 所有业务对象（User, Role, Notification等）都关联tenantId
- 跨租户操作严格禁止，违规作为CRITICAL级审计事件
- 资源池通过ResourcePool与Tenant关联实现资源隔离

### 2. 权限管理（RBAC + ABAC）
- 基于角色的访问控制（RBAC）：User → UserTenantRole → Role → Permissions
- 基于属性的访问控制（ABAC）：Permission条件字段支持细粒度控制
- 三级权限：菜单级（导航展示）→ 功能级（操作按钮）→ 数据行级（字段可见性）

### 3. 审计与安全
- 所有重要操作必须记录AuditLog（包括创建、修改、删除、登录等）
- CRITICAL级操作需要额外审批或即时告警
- 日志签名防篡改，长期冷存储
- 操作人信息（ID、名称、BU、角色）完整记录，便于追溯

### 4. 通知机制
- 支持多渠道投递（IN_APP、EMAIL、IM）
- 异步非阻塞，最多重试3次
- 用户可控制通知偏好、订阅类型
- 消息过期自动清理（30天默认）

### 5. 组织树与继承
- Tenant支持无限层级树形结构（CORP → BU → PROJECT）
- 禁用父租户自动级联禁用子租户及其用户
- 权限继承：自定义角色可继承预设角色权限上限
