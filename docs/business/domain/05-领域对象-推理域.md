# 05-领域对象 – 推理域

> **范围**：推理域负责中心端推理服务的版本与流量管理、边端服务器注册与模型下发、推理应用鉴权与有效期管理。
> **组织方式**：按聚合根分组。每个聚合根包含标识、核心属性、状态机、关联关系和子对象。

---

## 聚合根1：推理服务（InferenceService Aggregate）

**领域对象**：`InferenceService`
**标识**：`serviceId: UUID`

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `serviceId` | UUID | 推理服务唯一标识 |
| `name` | String | 服务名称 |
| `description` | String | 服务描述 |
| `modelId` | UUID | 关联的模型ID（引用Model） |
| `modelVersion` | String | 模型版本号 |
| `scope` | Enum | 使用范围：PLATFORM / TEAM |
| `authType` | Enum | 鉴权方式：NONE / API_KEY |
| `protocol` | Enum | 协议类型：HTTP / gRPC / WebSocket |
| `status` | Enum | 状态（见下方状态机） |
| `tenantId` | UUID | 所属租户ID |
| `createdBy` | UUID | 创建者ID |
| `createdAt` | Timestamp | 创建时间 |

**子对象**：`DeploymentVersion`

| 属性 | 类型 | 说明 |
|------|------|------|
| `versionId` | UUID | 部署版本ID |
| `tag` | String | 版本标签（如v1.0、canary） |
| `resourceConfig` | ResourceConfig | 资源配置（CPU/内存/GPU/副本数） |
| `scalingPolicy` | Enum | 弹性策略：NONE / HPA / SERVERLESS |
| `healthCheck` | HealthCheck | 健康检查（TCP / HTTP） |
| `trafficWeight` | Integer | 流量权重（0-100，多版本时所有版本求和为100） |
| `status` | Enum | 版本状态：DEPLOYING / RUNNING / STOPPED / FAILED |
| `createdAt` | Timestamp | 创建时间 |

**子对象**：`TrafficPolicy`

| 属性 | 类型 | 说明 |
|------|------|------|
| `policyId` | UUID | 策略ID |
| `versionWeights` | Map<UUID, Integer> | 版本→流量权重映射，totalWeight 必须等于100 |
| `grayRules` | List<Object> | 灰度发布规则（按Header/Cookie/百分比路由等） |
| `totalWeight` | Integer | 权重总和（必须等于100） |
| `updatedAt` | Timestamp | 更新时间 |

**子对象**：`HealthCheck`

| 属性 | 类型 | 说明 |
|------|------|------|
| `type` | Enum | 检查类型：TCP / HTTP |
| `path` | String | HTTP 检查路径（type=HTTP 时必填） |
| `intervalSeconds` | Integer | 检查间隔（秒） |
| `timeoutSeconds` | Integer | 超时时间（秒） |
| `failureThreshold` | Integer | 失败阈值（连续失败次数） |

**子对象**：`ResourceConfig`

| 属性 | 类型 | 说明 |
|------|------|------|
| `cpu` | String | CPU规格（如"4"表示4核） |
| `gpu` | String | GPU规格（如"1×T4"） |
| `memory` | String | 内存大小（如"8Gi"） |
| `replicas` | Integer | 副本数 |

**状态机**：
```
CREATED → DEPLOYING → RUNNING → STOPPED
              ↓          ↓
            FAILED    FAILED
```

- **CREATED**：服务已创建，未部署
- **DEPLOYING**：正在部署中
- **RUNNING**：服务正常运行
- **STOPPED**：服务已停止
- **FAILED**：部署或运行失败

**关联关系**：
- `N:1 → Model`：关联的推理模型
- `1:N → DeploymentVersion`：一个服务可有多个部署版本
- `1:1 → TrafficPolicy`：流量分发策略
- `1:N → InferenceApp`：被多个推理应用授权访问

**操作**：
- `deploy()`：部署新版本
- `scale()`：扩缩容（手动或基于策略自动触发）
- `rollback()`：回滚到上一稳定版本
- `stopService()`：停止服务
- `testAccess()`：在线测试调用接口

**业务规则引用**：
- **INF-001**：健康检查连续失败时自动回滚到上一稳定版本
- **INF-006**：连续3次部署失败后自动回滚并冻结后续部署

## 聚合根2：边端服务器（EdgeServer Aggregate）

**领域对象**：`EdgeServer`
**标识**：`serverId: UUID`

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `serverId` | UUID | 边端服务器唯一标识 |
| `name` | String | 服务器名称 |
| `location` | String | 部署地点（如"上海工厂A车间"） |
| `ipAddress` | String | IP地址 |
| `status` | Enum | 状态（见下方状态机） |
| `tenantId` | UUID | 所属租户ID |
| `registeredAt` | Timestamp | 注册时间 |
| `lastHeartbeatAt` | Timestamp | 最近心跳时间 |

**子对象**：`EdgeDeployment`

| 属性 | 类型 | 说明 |
|------|------|------|
| `deploymentId` | UUID | 边端部署ID |
| `serviceId` | UUID | 关联的推理服务ID |
| `modelId` | UUID | 下发的模型ID |
| `modelVersion` | String | 模型版本号 |
| `deployStatus` | Enum | 部署状态：PENDING / DOWNLOADING / RUNNING / FAILED |
| `syncStatus` | Enum | 同步状态：SYNCED / PENDING / FAILED |
| `deployedAt` | Timestamp | 部署时间 |
| `lastSyncAt` | Timestamp | 最近同步时间 |

**状态机**：
```
PENDING → ACTIVE → OFFLINE → ACTIVE（恢复）
              ↓
        DECOMMISSIONED
```

- **PENDING**：已注册，等待激活
- **ACTIVE**：在线运行中
- **OFFLINE**：离线（心跳超时），可恢复为 ACTIVE
- **DECOMMISSIONED**：已停用，不再可用

**关联关系**：
- `1:N → EdgeDeployment`：一台边端服务器可承载多个部署
- `N:1 → InferenceService`：每个部署关联到中心端推理服务（通过 EdgeDeployment.serviceId）
- `N:1 → Model`：每个部署关联具体的模型版本

**操作**：
- `register()`：注册边端服务器
- `deployModel()`：下发模型到边端（需审批）
- `syncStatus()`：同步边端状态与心跳
- `decommission()`：停用边端服务器

**业务规则引用**：
- **INF-003**：模型下发到边端需经过审批流程
- **INF-008**：模型下发完成后须进行完整性校验（哈希校验）

## 聚合根3：推理应用（InferenceApp Aggregate）

**领域对象**：`InferenceApp`
**标识**：`appId: UUID`

**核心属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `appId` | UUID | 推理应用唯一标识 |
| `appName` | String | 应用名称 |
| `appKey` | String | 应用Key（公开标识） |
| `secretKey` | String | 应用密钥（敏感，需加密存储） |
| `serviceIds` | List<UUID> | 授权访问的推理服务ID列表 |
| `ipWhitelist` | List<String> | IP白名单（可选，启用时仅白名单内的IP可调用） |
| `expiredAt` | Timestamp | 过期时间 |
| `status` | Enum | 状态：ACTIVE / EXPIRED / REVOKED |
| `createdBy` | UUID | 创建者ID |
| `tenantId` | UUID | 所属租户ID |
| `createdAt` | Timestamp | 创建时间 |

**状态机**：
```
ACTIVE → EXPIRED
   ↓
REVOKED
```

- **ACTIVE**：有效，可正常调用授权服务
- **EXPIRED**：已过期，调用被拒绝（可通过 renewExpiry 续期）
- **REVOKED**：已吊销，永久不可用

**关联关系**：
- `N:M → InferenceService`：一个应用可授权访问多个服务，一个服务可被多个应用调用
- `N:1 → Tenant`：归属租户

**操作**：
- `revokeApp()`：吊销应用
- `renewExpiry()`：续期有效期

## 跨聚合根关系总结

| 源聚合根 | 关联 | 目标聚合根 | 说明 |
|---------|------|-----------|------|
| InferenceService | N:1 | Model | 关联推理模型（跨域，模型域） |
| InferenceService | 1:N | DeploymentVersion | 多版本部署管理 |
| InferenceService | 1:1 | TrafficPolicy | 流量分发策略 |
| InferenceService | N:M | InferenceApp | 被推理应用授权调用 |
| EdgeServer | 1:N | EdgeDeployment | 一台边端承载多个部署 |
| EdgeServer | N:1 | InferenceService | 边端部署关联中心端服务 |
| EdgeServer | N:1 | Model | 边端部署关联模型版本（跨域，模型域） |
| InferenceApp | N:M | InferenceService | 授权调用关系 |
| InferenceApp | N:1 | Tenant | 归属租户（跨域，平台域） |
