# YFI工业AI小模型平台（SMP）— 领域模型索引

> **文档目的**：为SMP平台的领域建模提供统一的视图，指导开发人员理解系统的业务逻辑、数据流向、资源管理机制和横切能力。本索引按领域分层组织，每个领域文档遵循聚合根 → 领域对象 → 关联关系的结构化方式。

---

## 一、领域划分概览

SMP平台的核心功能根据业务职责划分为**4大领域**和**1个横切能力层**：

| 领域代码 | 领域名称 | 英文名 | 范围说明 | 核心职责 |
|---------|---------|-------|---------|---------|
| **DATA** | 数据域 | Data Domain | 数据源、数据加工、标注、增强 | 数据生命周期管理、数据质量 |
| **MODEL** | 模型域 | Model Domain | 模型训练、工程化、开发、推理 | 模型开发与管理 |
| **RESOURCE** | 资源域 | Resource Domain | 计算集群、存储、镜像、芯片 | 资源统一调度与分配 |
| **PLATFORM** | 平台域 | Platform Domain | 租户、权限、审计、安全 | 平台级横切能力 |
| **INFERENCE** | 推理域 | Inference Domain | 推理服务、边端管理、应用鉴权 | 推理服务生命周期与边端模型下发 |

## 二、各领域核心聚合根

### 数据域（DATA Domain）

| 聚合根 | 实体类 | 说明 |
|-------|-------|------|
| **数据源** | `DataSource` | 对接各类数据来源（DB、文件、流、API等） |
| **数据加工流水线** | `DataPipeline` | 定义数据处理算子链路 |
| **标注任务** | `AnnotationTask` | 管理数据标注全流程 |
| **数据集** | `Dataset` | 数据集版本、血缘、标签管理 |
| **标签模板** | `LabelTemplate` | 标注标签体系定义 |
| **数据增强任务** | `DataAugmentationJob` | 数据增强处理 |
| **数据集访问申请** | `AccessRequest` | 受限数据集访问审批（DAT-006） |
| **审批工单** | `ApprovalTicket` | 访问申请的完整审批流程记录 |

### 模型域（MODEL Domain）

| 聚合根 | 实体类 | 说明 |
|-------|-------|------|
| **调优任务** | `FineTuningJob` | 模型微调训练 |
| **模型** | `Model` | 模型版本、框架、任务类型管理 |
| **工程化任务** | `ModelEngineeringJob` | 量化、剪枝、蒸馏等优化 |
| **开发任务** | `DevelopmentTask` | Jupyter/VSCode/SSH开发环境 |
| **训练任务** | `TrainingJob` | 单机/分布式训练管理 |

### 资源域（RESOURCE Domain）

| 聚合根 | 实体类 | 说明 |
|-------|-------|------|
| **集群** | `Cluster` | 物理/逻辑计算集群 |
| **资源池** | `ResourcePool` | 资源隔离与共享策略 |
| **AI芯片** | `AIChip` | GPU/NPU设备管理 |
| **存储池** | `StoragePool` | 存储后端与容量管理 |
| **容器镜像** | `ContainerImage` | 开发/训练/推理镜像库 |

### 平台域（PLATFORM Domain）

| 聚合根 | 实体类 | 说明 |
|-------|-------|------|
| **租户** | `Tenant` | 集团/BU/项目三级多租户管理 |
| **用户** | `User` | 账号、认证、角色绑定 |
| **角色** | `Role` | RBAC权限策略与授权管理 |
| **操作审计日志** | `AuditLog` | 操作行为追踪与合规审计 |
| **通知消息** | `Notification` | 站内/邮件/Webhook通知 |

### 推理域（INFERENCE Domain）

| 聚合根 | 实体类 | 说明 |
|-------|-------|------|
| **推理服务** | `InferenceService` | 中心端推理服务版本与流量管理 |
| **边端服务器** | `EdgeServer` | 边端设备注册与模型下发管理 |
| **推理应用** | `InferenceApp` | API鉴权、IP白名单、有效期管理 |

横切能力场景详见 **04-系统能力场景清单.md**

## 四、关键概念说明

### 聚合根（Aggregate Root）
每个聚合根代表一个业务实体的生命周期管理单元。聚合根通过标识（ID）唯一确定，包含核心属性、状态机和关联关系。

### 状态机（State Machine）
描述聚合根在其生命周期内的状态转移规则，确保业务逻辑的正确性。

### 关联关系（Relations）
- **1:N**：一个聚合根对应多个子实体
- **N:1**：多个聚合根指向同一个关联对象
- **M:N**：通过关联表表示多对多关系

### 子对象（Value Objects）
聚合根内部的值对象，没有独立的标识，用于补充聚合根的属性。

## 五、跨域关键流程

### 数据 → 模型 → 资源

```
[DataSource] ──→ [DataPipeline] ──→ [Dataset]
                                        │
                                        ↓
                    [AnnotationTask]  [LabelTemplate]
                                        │
                        ┌───────────────┴───────────────┐
                        ↓                               ↓
               [FineTuningJob]                   [TrainingJob]
                        │                               │
                        └───────────────┬───────────────┘
                                        ↓
                                    [Model]
                                        │
                                        ↓
                            [ModelEngineeringJob]
                                        │
                        ┌───────────────┴───────────────┐
                        ↓                               ↓
              [DevelopmentTask]           [Cluster] / [ResourcePool]
                                                        │
                                                        ↓
                                              [AIChip] / [StoragePool]
```

> **Q-011 修正（2026-05-09）**：`FineTuningJob` 与 `TrainingJob` 是**并列**的两种训练路径，均以 `Dataset` 为输入、产出 `Model`，互不依赖。原图误将 `TrainingJob` 置于 `ModelEngineeringJob` 下游，已更正。

## 六、使用指南

1. **快速查询**：根据关键词（数据源、模型训练、资源调度等）找到对应领域文档
2. **设计参考**：新增功能时参考相关聚合根的属性、状态机、关联关系
3. **集成开发**：跨域功能需参考关键流程图，理解各领域的交互点
4. **扩展建议**：新聚合根添加前应在索引中声明，并补充完整的领域对象文档
